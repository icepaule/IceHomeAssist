Integrationen

Home Assistant wird durch Integrationen mit externen Diensten und Geräten verbunden. Das IceHomeAssist-System nutzt sowohl offizielle Integrationen als auch Custom Components über HACS (Home Assistant Community Store). Diese Seite dokumentiert alle verwendeten Integrationen mit Versionen, Beschreibungen und Setup-Hinweisen.

HACS (Home Assistant Community Store)

Was ist HACS? HACS ist ein Community-Store, der die Installation und Verwaltung von Custom Components, Themes und Lovelace-Karten ermöglicht. Es wird selbst als Custom Component installiert und bietet eine grafische Oberfläche in Home Assistant.

HACS installieren

# HACS herunterladen und installieren (im Container)
sudo docker exec -it homeassistant bash

# Innerhalb des Containers:
wget -O - https://get.hacs.xyz | bash -

# Danach Home Assistant neu starten
# und HACS über Einstellungen → Integrationen hinzufügen

HACS Custom Components im Detail

Alexa Media Player v5.11.0

Ermöglicht die vollständige Steuerung von Amazon Echo-Geräten direkt aus Home Assistant. Unterstützt Medienwiedergabe, Text-to-Speech (TTS), Routinen-Auslösung und Geräte-Gruppierung.

Haupt-Entity media_player.marcus_echo_show_8_2nd_gen
Gruppen-Entity media_player.uberall (Überall/Everywhere)
Setup-Hinweis: Für die Einrichtung ist 2FA auf dem Amazon-Konto erforderlich. Bei 500-Fehlern während des Setups handelt es sich in der Regel um Amazons Bot-Erkennung ("aamation" Challenge), nicht um HA-Bugs. Mehrere Versuche können nötig sein.

Divera Control v1.3.1

Integration für das Alarmierungssystem Divera 24/7, das vom THW (Technisches Hilfswerk) und Feuerwehren verwendet wird. Zeigt aktuelle Alarme, Einsatzdetails und Verfügbarkeiten an. 

# Konfiguration über die UI
# Einstellungen → Integrationen → Divera Control
# Benötigt: API-Key aus Divera 24/7

Dreame Vacuum v1.0.8

Vollständige Steuerung des Dreame Staubsaugerroboters. Unterstützt Start/Stop, Raum-spezifische Reinigung, Saugstärke, Wischmodus und Live-Karte.

Wird in Kombination mit dem Xiaomi Cloud Map Extractor (v2.2.5) verwendet, um die Reinigungskarte im Dashboard anzuzeigen.

DWD Wetter-Integrationen

Drei separate HACS-Komponenten für umfassende Wetterdaten vom Deutschen Wetterdienst:

Komponente Version Funktion
DWD 2025.12.1 Allgemeine Wetterdaten: Temperatur, Luftfeuchtigkeit, Wind, Wetterlage
DWD Precipitation 2025.7.0b1 Aktueller Niederschlagssensor (Radar-basiert)
DWD Precipitation Forecast v0.2.0 Niederschlagsvorhersage für die nächsten Stunden

Fritz!Box VPN v0.7.0

Überwacht den Status der VPN-Verbindungen auf der Fritz!Box. Zeigt an, welche VPN-Tunnel aktiv sind und ermöglicht die Überwachung der Verbindungsstabilität.

Meross Integrationen

Für Meross-Geräte werden zwei Komponenten parallel verwendet:

Komponente Version Vorteil
Meross LAN v5.8.0 Lokale Steuerung ohne Cloud – schnellere Reaktionszeiten, funktioniert ohne Internet
Meross Cloud v1.3.12 Cloud-basierte Steuerung als Fallback, Firmware-Updates, Ersteinrichtung
Empfehlung: Meross LAN sollte bevorzugt werden, da es lokal arbeitet. Meross Cloud dient als Backup und für die initiale Geräte-Erkennung.

PetKit v0.1.14

Integration für PetKit-Geräte wie Futterautomaten und Trinkbrunnen. Ermöglicht die Überwachung des Futterstands, der Wasserqualität und des Fütterungszeitplans. Wird im Charly-Dashboard verwendet.

Xiaomi Cloud Map Extractor v2.2.5

Extrahiert die Reinigungskarte des Staubsaugers aus der Xiaomi/Dreame Cloud und stellt sie als Kamera-Entity in Home Assistant bereit. Wird im Staubsauger-Dashboard als Live-Karte angezeigt. 

# Beispiel: Kamera-Entity für die Staubsauger-Karte
camera:
  - platform: xiaomi_cloud_map_extractor
    name: "Staubsauger Karte"
    host: !secret dreame_host
    token: !secret dreame_token
    username: !secret xiaomi_username
    password: !secret xiaomi_password
    draw:
      - all

Offizielle Integrationen

MQTT

Die offizielle MQTT-Integration verbindet Home Assistant mit dem Mosquitto Broker. Alle Tasmota-Geräte (Energiesensoren, WiFi-Buttons) kommunizieren über MQTT.

Siehe MQTT-Seite für ausführliche Dokumentation.

Tibber

Tibber ist der dynamische Stromanbieter. Die Integration liefert stündliche Strompreise, Verbrauchsdaten und ermöglicht die Optimierung des Energieverbrauchs basierend auf dem aktuellen Strompreis.

Siehe Energie-Seite für ausführliche Dokumentation.

Sony Bravia TV

Steuerung des Sony Bravia Android TV. Ermöglicht Ein/Ausschalten, Lautstärke, App-Start und Quellenwahl über Home Assistant.

Fritz!Box (TR-064)

Offizielle Integration für den Fritz!Box Router. Bietet Geräte-Tracking (wer ist im Netzwerk), Anrufüberwachung, DSL-Verbindungsstatus und Smart-Home-Geräte-Steuerung.

UniFi Network

Integration für Ubiquiti UniFi Netzwerk-Equipment. Ermöglicht das Monitoring von Switches, Access Points und verbundenen Clients. Wird im Netzwerk-Dashboard verwendet.

HACS-Komponenten verwalten

Update-Prozess

# 1. HACS öffnen (Web-UI: Einstellungen → HACS)
# 2. Auf "Updates verfügbar" prüfen
# 3. Gewünschte Komponente auswählen → "Aktualisieren"
# 4. Home Assistant neu starten

# Alternativ: Manuelles Update über die Kommandozeile
cd /config/custom_components/
ls -la  # Installierte Komponenten anzeigen
Update-Hinweis: Vor dem Update einer HACS-Komponente immer die Release-Notes lesen! Breaking Changes können die Konfiguration invalidieren. Ein Backup vor dem Update ist empfehlenswert.
