MQTT - Message Queuing Telemetry Transport

MQTT ist das zentrale Kommunikationsprotokoll in unserem Smart Home. Es verbindet alle Tasmota-basierten Energiesensoren, WiFi-Buttons und weitere IoT-Geräte mit Home Assistant. Der Mosquitto Broker läuft als Addon direkt in Home Assistant und verwaltet sämtliche Nachrichten zwischen den Geräten.

Mosquitto Addon - Setup und Konfiguration

Der Mosquitto MQTT Broker ist als offizielles Home Assistant Addon installiert. Er ist die zentrale Nachrichtenvermittlung für alle MQTT-fähigen Geräte im Netzwerk.

Installation

Das Mosquitto Addon wird über den Add-on Store in Home Assistant installiert:

  1. Einstellungen → Add-ons → Add-on Store
  2. Nach Mosquitto broker suchen
  3. Installieren klicken und warten bis die Installation abgeschlossen ist
  4. Unter Konfiguration die gewünschten Einstellungen vornehmen
  5. Starten und Bei Systemstart starten aktivieren

Addon-Konfiguration

Die Mosquitto-Konfiguration erfolgt über die Addon-Oberfläche. Hier die wichtigsten Einstellungen: 

# Mosquitto Addon Konfiguration
logins:
  - username: mqtt_user
    password: "!secret mqtt_password"
require_certificate: false
certfile: fullchain.pem
keyfile: privkey.pem
customize:
  active: false
  folder: mosquitto

Benutzer anlegen

Für die MQTT-Authentifizierung wird ein dedizierter Benutzer in Home Assistant angelegt:

  1. Einstellungen → Personen → Benutzer
  2. Neuen Benutzer erstellen (z.B. mqtt_user)
  3. Diesem Benutzer keine Administrator-Rechte geben
  4. Benutzername und Passwort in secrets.yaml hinterlegen

Ports und Protokolle

Port Protokoll Beschreibung
1883 MQTT (unverschlüsselt) Standard-Port für lokale Geräte
8883 MQTT over TLS Verschlüsselte Verbindung (optional)
1884 WebSocket Für Browser-basierte Clients

Tasmota Energiesensoren

Im Netzwerk befinden sich sechs Tasmota-geflashte Steckdosen bzw. Energiemessgeräte, die über MQTT ihre Messwerte an Home Assistant senden. Jedes Gerät meldet regelmäßig Spannung, Strom, Leistung und den kumulierten Energieverbrauch.

Geräteübersicht

Gerät MQTT Topic-Prefix Beschreibung Messbereich
Strom-PV Strom-PV Photovoltaik-Anlage Einspeise-/Bezugsmessung 0-3680W
Strom-Server Strom-Server Server-Rack und NUC im Keller 0-3680W
Strom-Trockner Strom-Trockner Wäschetrockner im Hauswirtschaftsraum 0-3680W
Strom-Wasch Strom-Wasch Waschmaschine im Hauswirtschaftsraum 0-3680W

Tasmota MQTT-Konfiguration

Jedes Tasmota-Gerät wird über die Web-Oberfläche (erreichbar unter der jeweiligen IP) konfiguriert. Die MQTT-Einstellungen müssen wie folgt gesetzt werden: 

# Tasmota MQTT Einstellungen (pro Gerät)
Host:     homeassistant.local   (oder IP des HA Servers)
Port:     1883
Client:   DVES_%06X            (automatisch generiert)
User:     mqtt_user
Password: [aus secrets.yaml]
Topic:    [Gerätename, z.B. Strom-PV]
Full Topic: %prefix%/%topic%/

Nach dem Speichern startet das Gerät neu und verbindet sich automatisch mit dem Mosquitto Broker. Im Tasmota-Log sollte die Meldung MQT: Connected erscheinen.

MQTT Topic-Struktur

Tasmota verwendet eine standardisierte Topic-Struktur mit drei Präfixen. Das Verständnis dieser Struktur ist entscheidend für die korrekte Konfiguration in Home Assistant.

Telemetrie-Topics (tele/)

Telemetrie-Nachrichten werden periodisch gesendet (Standard: alle 300 Sekunden) und enthalten die aktuellen Sensorwerte. 

# Telemetrie-Topic mit Sensorwerten (alle 300s)
Topic: tele/Strom-PV/SENSOR

Payload (JSON):
{
  "Time": "2026-02-07T14:30:00",
  "ENERGY": {
    "TotalStartTime": "2024-01-15T00:00:00",
    "Total": 1523.456,
    "Yesterday": 4.231,
    "Today": 2.876,
    "Power": 187,
    "ApparentPower": 192,
    "ReactivePower": 45,
    "Factor": 0.97,
    "Voltage": 233,
    "Current": 0.824
  }
}

# Weitere Telemetrie-Topics:
tele/Strom-PV/STATE          # Gerätestatus (Uptime, WiFi-Signal, etc.)
tele/Strom-PV/LWT            # Last Will and Testament (Online/Offline)

Status-Topics (stat/)

Status-Topics enthalten die Antworten auf Befehle und den aktuellen Schaltzustand des Geräts. 

# Schaltzustand des Geräts
Topic: stat/Strom-PV/POWER
Payload: ON oder OFF

# Ergebnis eines Befehls
Topic: stat/Strom-PV/RESULT
Payload: {"POWER":"ON"}

# WiFi-Button Ergebnis
Topic: stat/wifibutton1/RESULT
Payload: {"Button1":{"Action":"SINGLE"}}
         {"Button2":{"Action":"SINGLE"}}
         {"Button3":{"Action":"SINGLE"}}
         {"Button4":{"Action":"0xD26F68"}}   # RF Button
         {"Button5":{"Action":"0xDD21A8"}}   # RF Button

Kommando-Topics (cmnd/)

Über Kommando-Topics können Geräte gesteuert und konfiguriert werden. 

# Gerät ein-/ausschalten
Topic: cmnd/Strom-PV/POWER
Payload: ON / OFF / TOGGLE

# Telemetrie-Intervall ändern (in Sekunden)
Topic: cmnd/Strom-PV/TelePeriod
Payload: 60    (Minimum: 10, Maximum: 3600)

# Status abfragen
Topic: cmnd/Strom-PV/Status
Payload: 0     (0 = alle Statusmeldungen)

# Energiezähler zurücksetzen
Topic: cmnd/Strom-PV/EnergyReset
Payload: 0     (Total auf 0 setzen)

Topic-Übersicht pro Gerät

Prefix Topic Richtung Beschreibung
tele/ tele/DEVICE/SENSOR Gerät → HA Periodische Sensorwerte
tele/ tele/DEVICE/STATE Gerät → HA Gerätestatus
tele/ tele/DEVICE/LWT Gerät → HA Online/Offline Status
stat/ stat/DEVICE/POWER Gerät → HA Aktueller Schaltzustand
stat/ stat/DEVICE/RESULT Gerät → HA Befehlsantwort
cmnd/ cmnd/DEVICE/POWER HA → Gerät Schaltbefehl

WiFi-Button MQTT-Konfiguration

Der WiFi-Button basiert auf einem ESP32-S3 mit Tasmota-Firmware. Er verfügt über drei physische Taster und empfängt zusätzlich zwei RF-Signale über einen 433MHz-Empfänger. Alle Button-Events werden über das Topic stat/wifibutton1/RESULT veröffentlicht.

Button-Zuordnung

Button Typ GPIO / RF-Code MQTT Payload
Button1 Physisch GPIO42 {"Button1":{"Action":"SINGLE"}}
Button2 Physisch GPIO41 {"Button2":{"Action":"SINGLE"}}
Button3 Physisch GPIO40 {"Button3":{"Action":"SINGLE"}}
Button4 RF 433MHz 0xD26F68 {"Button4":{"Action":"0xD26F68"}}
Button5 RF 433MHz 0xDD21A8 {"Button5":{"Action":"0xDD21A8"}}

Die Aktionszuordnung der Buttons wird über input_select Helper in Home Assistant konfiguriert. Die eigentliche Automatisierung erfolgt über Node-RED (siehe Node-RED Dokumentation). Die Home Assistant Automationen sind mit initial_state: false deaktiviert, da Node-RED die Steuerung übernimmt.

Die MQTT-Integration wird in der configuration.yaml bzw. in den Package-Dateien konfiguriert. Hier werden Sensoren, Schalter und weitere Entities für die Tasmota-Geräte definiert.

MQTT Debugging und Monitoring

Für die Fehlersuche und das Monitoring der MQTT-Kommunikation stehen verschiedene Werkzeuge zur Verfügung.

MQTT Explorer (Desktop-Tool)

Der MQTT Explorer ist ein grafisches Tool zum Beobachten aller MQTT-Topics in Echtzeit. Er zeigt die Topic-Hierarchie, aktuelle Werte und den Verlauf an. 

# Verbindungseinstellungen für MQTT Explorer:
Host:     IP-Adresse des Home Assistant Servers
Port:     1883
Username: mqtt_user
Password: [aus secrets.yaml]

mosquitto_sub (Kommandozeile)

Mit dem Kommandozeilen-Tool mosquitto_sub können Topics direkt abonniert werden: 

# Alle Topics eines Geräts mithören
mosquitto_sub -h homeassistant.local -u mqtt_user -P "password" \
  -t "tele/Strom-PV/#" -v

# Alle Telemetrie-Topics aller Geräte
mosquitto_sub -h homeassistant.local -u mqtt_user -P "password" \
  -t "tele/+/SENSOR" -v

# WiFi-Button Events mithören
mosquitto_sub -h homeassistant.local -u mqtt_user -P "password" \
  -t "stat/wifibutton1/RESULT" -v

# Alle Topics (Vorsicht: sehr viele Nachrichten!)
mosquitto_sub -h homeassistant.local -u mqtt_user -P "password" \
  -t "#" -v

mosquitto_pub (Befehle senden)

# Gerät einschalten
mosquitto_pub -h homeassistant.local -u mqtt_user -P "password" \
  -t "cmnd/Strom-PV/POWER" -m "ON"

# Status abfragen
mosquitto_pub -h homeassistant.local -u mqtt_user -P "password" \
  -t "cmnd/Strom-PV/Status" -m "0"

# Telemetrie sofort auslösen
mosquitto_pub -h homeassistant.local -u mqtt_user -P "password" \
  -t "cmnd/Strom-PV/TelePeriod" -m "10"

Home Assistant MQTT Tool

Home Assistant bietet unter Einstellungen → Geräte & Dienste → MQTT → Konfigurieren ein integriertes Tool zum Abhören und Senden von MQTT-Nachrichten. Dies ist besonders nützlich für schnelle Tests ohne externe Tools.

Tasmota Auto-Discovery

Tasmota unterstützt die automatische Erkennung über MQTT Discovery. Wenn aktiviert, registrieren sich Tasmota-Geräte automatisch in Home Assistant, ohne dass manuelle YAML-Konfiguration nötig ist. 

# Tasmota Discovery aktivieren (in Tasmota Konsole):
SetOption19 0    # Native Discovery deaktivieren (alt)
SetOption73 1    # MQTT Discovery aktivieren (neu, empfohlen)

# Home Assistant erkennt das Gerät automatisch unter:
# Einstellungen > Geräte & Dienste > MQTT > Geräte

Hinweis: In diesem Setup wird die manuelle YAML-Konfiguration bevorzugt, da sie mehr Kontrolle über Entity-Namen, Unique-IDs und Verfügbarkeits-Topics bietet. Die Auto-Discovery kann jedoch für neue Geräte als Ausgangspunkt genutzt werden.

Häufige Probleme

Problem Ursache Lösung
Sensor zeigt unavailable Gerät offline oder LWT-Topic nicht konfiguriert Tasmota-Gerät prüfen, WLAN-Verbindung sicherstellen
Werte aktualisieren sich nicht TelePeriod zu hoch oder Topic falsch cmnd/DEVICE/TelePeriod 60 setzen
Doppelte Entities Auto-Discovery und manuelle Konfiguration gleichzeitig Entweder Discovery oder YAML verwenden
Verbindung wird ständig getrennt Gleiche Client-ID bei mehreren Geräten Eindeutige Client-IDs in Tasmota setzen
Authentifizierungsfehler Falscher Benutzername oder Passwort Zugangsdaten in Tasmota und Mosquitto prüfen
