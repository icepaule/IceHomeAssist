Energiemanagement

Das Energiemanagement ist eine der Kernfunktionen unseres Smart Homes. Über sechs Tasmota-basierte Energiesensoren wird der Stromverbrauch einzelner Geräte und Bereiche überwacht. In Kombination mit der Tibber-Integration für dynamische Strompreise und der PV-Anlage ergibt sich ein umfassendes Energiemonitoring mit automatisierter Verbrauchsoptimierung.

PV-Anlage Monitoring

Die Photovoltaik-Anlage wird über den Tasmota-Sensor Strom-PV überwacht. Der Sensor misst die aktuelle Einspeiseleistung, den Tagesertrag und den Gesamtertrag seit Installation.

Sensoren für die PV-Anlage

Entity Beschreibung Einheit
sensor.pv_leistung Aktuelle Leistung der PV-Anlage W
sensor.pv_energie_heute Heutiger Ertrag kWh
sensor.pv_energie_gesamt Gesamtertrag seit Inbetriebnahme kWh
sensor.pv_spannung Netzspannung am PV-Einspeisepunkt V
sensor.pv_strom Aktueller Strom A

PV-Ertragsprognose

Für die Ertragsprognose wird der aktuelle Leistungswert mit den Wetterdaten (DWD-Integration) kombiniert. Bewölkungsgrad und Sonnenscheindauer erlauben eine grobe Abschätzung des zu erwartenden Tagesertrags. 

# Template-Sensor für PV-Status
template:
  - sensor:
      - name: "PV Status"
        state: >
          {% set power = states('sensor.pv_leistung') | float(0) %}
          {% if power > 500 %}
            Hohe Produktion
          {% elif power > 100 %}
            Mittlere Produktion
          {% elif power > 10 %}
            Geringe Produktion
          {% else %}
            Keine Produktion
          {% endif %}
        icon: >
          {% set power = states('sensor.pv_leistung') | float(0) %}
          {% if power > 100 %}
            mdi:solar-power
          {% else %}
            mdi:solar-power-variant-outline
          {% endif %}

Tibber Integration

Tibber ist unser Stromanbieter mit dynamischen, stündlich wechselnden Strompreisen. Die Integration liefert den aktuellen Strompreis, Preisprognosen für die nächsten Stunden und den täglichen Verbrauch. Diese Daten sind die Grundlage für die preisbasierte Steuerung von Geräten.

Tibber Sensoren

Entity Beschreibung Aktualisierung
sensor.electricity_price Aktueller Strompreis inkl. aller Abgaben Stündlich
sensor.tibber_price_level Preisniveau (VERY_CHEAP, CHEAP, NORMAL, EXPENSIVE, VERY_EXPENSIVE) Stündlich
sensor.tibber_daily_cost Tageskosten Täglich
sensor.tibber_monthly_cost Monatskosten Monatlich

Tibber-Konfiguration

Die Tibber-Integration wird über die Home Assistant UI eingerichtet:

  1. Einstellungen → Geräte & Dienste → Integration hinzufügen
  2. Nach Tibber suchen und auswählen
  3. Den persönlichen API-Token eingeben (erhältlich unter developer.tibber.com)
  4. Die Integration erstellt automatisch die Preis-Sensoren

Tibber Preislevel-Template

# Template-Sensor: Strompreis-Ampel
template:
  - sensor:
      - name: "Strompreis Ampel"
        state: >
          {% set level = states('sensor.tibber_price_level') %}
          {% if level in ['VERY_CHEAP', 'CHEAP'] %}
            grün
          {% elif level == 'NORMAL' %}
            gelb
          {% else %}
            rot
          {% endif %}
        icon: >
          {% set level = states('sensor.tibber_price_level') %}
          {% if level in ['VERY_CHEAP', 'CHEAP'] %}
            mdi:circle-slice-8
          {% elif level == 'NORMAL' %}
            mdi:circle-slice-4
          {% else %}
            mdi:alert-circle
          {% endif %}

Strompreis-basierte Automatisierungen

Durch die Kombination von Tibber-Strompreisen und schaltbaren Tasmota-Steckdosen können energieintensive Geräte automatisch in Niedrigpreiszeiten betrieben werden.

Waschmaschine bei günstigem Strompreis starten

# automations.yaml
- alias: "Waschmaschine - Günstig starten"
  description: "Schaltet die Waschmaschine frei, wenn der Strompreis günstig ist"
  trigger:
    - platform: state
      entity_id: sensor.tibber_price_level
      to:
        - "VERY_CHEAP"
        - "CHEAP"
  condition:
    - condition: state
      entity_id: input_boolean.waschmaschine_auto
      state: "on"
    - condition: time
      after: "06:00:00"
      before: "22:00:00"
  action:
    - service: switch.turn_on
      target:
        entity_id: switch.waschmaschine_steckdose
    - service: notify.alexa_media_marcus_echo_show_8_2nd_gen
      data:
        message: "Der Strompreis ist jetzt günstig. Die Waschmaschine wurde freigegeben."
        data:
          type: tts

Trockner bei teurem Strom sperren

- alias: "Trockner - Teuren Strom vermeiden"
  description: "Warnt, wenn der Trockner bei hohem Strompreis läuft"
  trigger:
    - platform: numeric_state
      entity_id: sensor.trockner_leistung
      above: 100
  condition:
    - condition: state
      entity_id: sensor.tibber_price_level
      state:
        - "EXPENSIVE"
        - "VERY_EXPENSIVE"
  action:
    - service: notify.alexa_media_uberall
      data:
        message: >
          Achtung! Der Trockner läuft gerade und der Strompreis ist sehr hoch.
          Aktueller Preis: {{ states('sensor.electricity_price') }} Cent pro Kilowattstunde.
        data:
          type: tts

PV-Überschuss nutzen

- alias: "PV-Überschuss - Geräte einschalten"
  description: "Schaltet Verbraucher ein, wenn PV-Überschuss vorhanden ist"
  trigger:
    - platform: numeric_state
      entity_id: sensor.pv_leistung
      above: 500
      for:
        minutes: 5
  condition:
    - condition: state
      entity_id: input_boolean.pv_optimierung
      state: "on"
  action:
    - service: switch.turn_on
      target:
        entity_id: switch.waschmaschine_steckdose
    - service: notify.alexa_media_marcus_echo_show_8_2nd_gen
      data:
        message: "PV-Überschuss erkannt. Verbraucher wurden eingeschaltet."
        data:
          type: tts

Energie Dashboard

Das Home Assistant Energie Dashboard bietet eine zentrale Übersicht über Stromproduktion, Verbrauch und Kosten. Es nutzt die state_class: total_increasing-Sensoren der Tasmota-Geräte.

Dashboard-Konfiguration

Das Energie Dashboard wird unter Einstellungen → Dashboards → Energie konfiguriert:

Stromproduktion (Solar)

  • Quelle: sensor.pv_energie_gesamt
  • Typ: Solar-Produktion

Stromverbrauch (Individuelle Geräte)

  • sensor.server_energie_gesamt - Server
  • sensor.trockner_energie_gesamt - Trockner
  • sensor.waschmaschine_energie_gesamt - Waschmaschine
  • sensor.kueche_energie_gesamt - Küche
  • sensor.arbeitszimmer_energie_gesamt - Arbeitszimmer

Strompreis

  • Quelle: Tibber Integration
  • Entity: sensor.electricity_price
  • Automatische Berechnung der Kosten pro Gerät

Utility Meters

Utility Meters in Home Assistant ermöglichen die Erfassung des Energieverbrauchs in definierten Zeiträumen (täglich, wöchentlich, monatlich, jährlich). Sie basieren auf den kumulierten Energiewerten der Tasmota-Sensoren.

utility_meters.yaml

    source: sensor.pv_energie_gesamt
    name: "PV Energie Täglich"
    cycle: daily

  server_energy_daily:
    source: sensor.server_energie_gesamt
    name: "Server Energie Täglich"
    cycle: daily

  trockner_energy_daily:
    source: sensor.trockner_energie_gesamt
    name: "Trockner Energie Täglich"
    cycle: daily

  waschmaschine_energy_daily:
    source: sensor.waschmaschine_energie_gesamt

# configuration.yaml
utility_meter: !include utility_meters.yaml

Energy Package

Das Energy Package fasst alle energiebezogenen Konfigurationen in einer Datei zusammen. Es enthält Template-Sensoren für berechnete Werte, Input-Booleans für die Automatisierungssteuerung und Hilfsentitäten.

packages/energy.yaml

# packages/energy.yaml
# ========================================
# Energie-Paket: Template-Sensoren und Helfer
# ========================================

# Input Booleans für Automatisierungssteuerung
input_boolean:
  waschmaschine_auto:
    name: "Waschmaschine Automatik"
    icon: mdi:washing-machine
  trockner_auto:
    name: "Trockner Automatik"
    icon: mdi:tumble-dryer
  pv_optimierung:
    name: "PV-Optimierung aktiv"
    icon: mdi:solar-power

# Template-Sensoren für berechnete Werte
template:
  - sensor:
      # Gesamtverbrauch aller überwachten Geräte
      - name: "Gesamtverbrauch Aktuell"
        unit_of_measurement: "W"
        device_class: power
        state_class: measurement
        state: >
          {{ [
            states('sensor.server_leistung') | float(0),
            states('sensor.trockner_leistung') | float(0),
            states('sensor.waschmaschine_leistung') | float(0),
            states('sensor.kueche_leistung') | float(0),
            states('sensor.arbeitszimmer_leistung') | float(0)
          ] | sum | round(1) }}

      # Eigenverbrauchsquote
      - name: "PV Eigenverbrauch"
        unit_of_measurement: "%"
        state: >
          {% set pv = states('sensor.pv_leistung') | float(0) %}
          {% set verbrauch = states('sensor.gesamtverbrauch_aktuell') | float(0) %}
          {% if pv > 0 %}
            {{ ([verbrauch / pv * 100, 100] | min) | round(1) }}
          {% else %}
            0
          {% endif %}

      # Nettobezug / Einspeisung
      - name: "Nettostrom"
        unit_of_measurement: "W"
        state: >
          {% set pv = states('sensor.pv_leistung') | float(0) %}
          {% set verbrauch = states('sensor.gesamtverbrauch_aktuell') | float(0) %}
          {{ (verbrauch - pv) | round(1) }}
        icon: >
          {% set netto = this.state | float(0) %}
          {% if netto > 0 %}
            mdi:transmission-tower-import
          {% else %}
            mdi:transmission-tower-export
          {% endif %}

      # Stromkosten aktuell (pro Stunde hochgerechnet)
      - name: "Aktuelle Stromkosten"
        unit_of_measurement: "ct/h"
        state: >
          {% set verbrauch_kw = states('sensor.gesamtverbrauch_aktuell') | float(0) / 1000 %}
          {% set preis = states('sensor.electricity_price') | float(0) %}
          {{ (verbrauch_kw * preis) | round(2) }}

      # Gerätestatus basierend auf Leistung
      - name: "Trockner Status"
        state: >
          {% set power = states('sensor.trockner_leistung') | float(0) %}
          {% if power > 1000 %}
            Heizt
          {% elif power > 100 %}
            Trocknet
          {% elif power > 5 %}
            Standby
          {% else %}
            Aus
          {% endif %}
        icon: mdi:tumble-dryer

      - name: "Waschmaschine Status"
        state: >
          {% set power = states('sensor.waschmaschine_leistung') | float(0) %}
          {% if power > 1500 %}
            Heizt
          {% elif power > 100 %}
            Wäscht
          {% elif power > 5 %}
            Standby
          {% else %}
            Aus
          {% endif %}
        icon: mdi:washing-machine

  # Binäre Sensoren
  - binary_sensor:
      - name: "Trockner läuft"
        state: >
          {{ states('sensor.trockner_leistung') | float(0) > 10 }}
        device_class: running

      - name: "Waschmaschine läuft"
        state: >
          {{ states('sensor.waschmaschine_leistung') | float(0) > 10 }}
        device_class: running

      - name: "PV produziert"
        state: >
          {{ states('sensor.pv_leistung') | float(0) > 10 }}
        device_class: power

      - name: "Strompreis günstig"
        state: >
          {{ states('sensor.tibber_price_level') in ['VERY_CHEAP', 'CHEAP'] }}

Benachrichtigungen

Energiebezogene Benachrichtigungen werden über Alexa (Text-to-Speech) und die Home Assistant Companion App gesendet.

Waschmaschine/Trockner fertig

- alias: "Waschmaschine fertig"
  trigger:
    - platform: state
      entity_id: binary_sensor.waschmaschine_lauft
      from: "on"
      to: "off"
      for:
        minutes: 2
  action:
    - service: notify.alexa_media_uberall
      data:
        message: "Die Waschmaschine ist fertig. Bitte die Wäsche entnehmen."
        data:
          type: tts

- alias: "Trockner fertig"
  trigger:
    - platform: state
      entity_id: binary_sensor.trockner_lauft
      from: "on"
      to: "off"
      for:
        minutes: 2
  action:
    - service: notify.alexa_media_uberall
      data:
        message: "Der Trockner ist fertig."
        data:
          type: tts

Tipps zur Energieoptimierung

  • Große Verbraucher in Billigstunden: Waschmaschine und Trockner möglichst dann laufen lassen, wenn der Tibber-Preis auf VERY_CHEAP steht.
  • PV-Überschuss nutzen: Bei hoher PV-Produktion automatisch Geräte einschalten.
  • Server-Verbrauch im Blick: Der Server läuft 24/7 - ein dauerhaft hoher Verbrauch deutet auf unnötige Last hin.
  • Standby vermeiden: Über die schaltbaren Steckdosen können Geräte komplett vom Netz getrennt werden.
  • Historische Analyse: Über das Energie Dashboard lassen sich Verbrauchsmuster erkennen und optimieren.
