Disaster Recovery - Wiederherstellung

Diese Seite beschreibt den vollständigen Wiederherstellungsprozess der Home Assistant Installation im Falle eines Totalausfalls. Das restore.sh Script automatisiert den Großteil der Arbeit. Es kann mit einem einzigen Befehl direkt von GitHub ausgeführt werden und stellt die gesamte Konfiguration auf einem frischen Debian-System wieder her.

Schnellstart: Ein-Befehl-Wiederherstellung

Im Notfall kann die gesamte Wiederherstellung mit einem einzigen Befehl gestartet werden. Voraussetzung ist ein frisches Debian-System mit Internetzugang und curl. 

# Komplette Wiederherstellung mit einem Befehl:
curl -sSL https://raw.githubusercontent.com/icepaule/IceHomeAssist/main/restore.sh | sudo bash

Hinweis: Das Script benötigt Root-Rechte (sudo), da es System-Pakete installiert, Docker einrichtet und Verzeichnisse unter /var/lib/homeassistant/ erstellt.

Voraussetzungen

  • Frisches Debian 12 (Bookworm) oder neuer auf dem NUC
  • Internetverbindung für Downloads
  • curl muss installiert sein (apt install curl)
  • Mindestens 20 GB freier Speicherplatz
  • Der Benutzer muss sudo-Rechte haben

restore.sh - Die 7 Phasen

Das Wiederherstellungs-Script durchläuft sieben aufeinander aufbauende Phasen. Jede Phase wird mit einer Statusmeldung angezeigt und kann bei Fehler einzeln wiederholt werden.

Phase 1: System-Pakete installieren

# Phase 1: Grundlegende Pakete
echo "=== Phase 1/7: System-Pakete installieren ==="
apt-get update
apt-get install -y \
    apt-transport-https \
    ca-certificates \
    curl \
    gnupg \
    lsb-release \
    git \
    jq \
    mosquitto-clients \
    snmp \
    snmp-mibs-downloader \
    python3 \
    python3-pip \
    python3-venv

Diese Phase stellt sicher, dass alle benötigten Systempakete vorhanden sind. Dazu gehören Git für das Klonen des Repositories, MQTT-Client-Tools für Debugging, SNMP-Tools für das Switch-Monitoring und Python für die eigenen Scripts.

Phase 2: Docker installieren

# Phase 2: Docker Engine
echo "=== Phase 2/7: Docker installieren ==="
curl -fsSL https://download.docker.com/linux/debian/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] \
  https://download.docker.com/linux/debian $(lsb_release -cs) stable" > /etc/apt/sources.list.d/docker.list
apt-get update
apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Docker Service starten und aktivieren
systemctl enable docker
systemctl start docker

Docker wird aus dem offiziellen Docker-Repository installiert. Die Docker Compose Plugin wird ebenfalls installiert, um den Container über eine docker-compose.yml verwalten zu können.

Phase 3: Verzeichnisstruktur erstellen

# Phase 3: Verzeichnisse
echo "=== Phase 3/7: Verzeichnisstruktur erstellen ==="
mkdir -p /var/lib/homeassistant/homeassistant
mkdir -p /var/lib/homeassistant/homeassistant/packages
mkdir -p /var/lib/homeassistant/homeassistant/dashboards
mkdir -p /var/lib/homeassistant/homeassistant/automations
mkdir -p /var/lib/homeassistant/homeassistant/templates
mkdir -p /var/lib/homeassistant/homeassistant/scripts
mkdir -p /var/lib/homeassistant/homeassistant/www
mkdir -p /var/lib/homeassistant/addon_configs/a0d7b954_nodered

Die Verzeichnisstruktur entspricht exakt dem Layout, das Home Assistant erwartet. Der Pfad /var/lib/homeassistant/homeassistant auf dem Host wird später als /config in den Docker-Container gemountet.

Phase 4: Git Repository klonen

# Phase 4: Repository
echo "=== Phase 4/7: Git Repository klonen ==="
cd /home/mpauli
if [ -d "IceHomeAssist" ]; then
    echo "Repository existiert bereits, aktualisiere..."
    cd IceHomeAssist && git pull
else
    git clone https://github.com/icepaule/IceHomeAssist.git
fi

Das Git-Repository enthält die gesamte YAML-Konfiguration, Packages, Dashboard-Definitionen, Automatisierungen und Scripts. Es enthält bewusst keine sensiblen Daten (siehe Sicherheit).

Phase 5: Konfiguration kopieren

# Phase 5: Konfiguration verteilen
echo "=== Phase 5/7: Konfiguration kopieren ==="
HA_CONFIG="/var/lib/homeassistant/homeassistant"
REPO="/home/mpauli/IceHomeAssist"

# Hauptkonfiguration
cp "$REPO/configuration.yaml" "$HA_CONFIG/"
cp "$REPO/automations.yaml" "$HA_CONFIG/" 2>/dev/null || true
cp "$REPO/scripts.yaml" "$HA_CONFIG/" 2>/dev/null || true
cp "$REPO/scenes.yaml" "$HA_CONFIG/" 2>/dev/null || true
cp "$REPO/utility_meters.yaml" "$HA_CONFIG/" 2>/dev/null || true

# Packages
cp -r "$REPO/packages/"* "$HA_CONFIG/packages/" 2>/dev/null || true

# Dashboards
cp -r "$REPO/dashboards/"* "$HA_CONFIG/dashboards/" 2>/dev/null || true

# Templates
cp -r "$REPO/templates/"* "$HA_CONFIG/templates/" 2>/dev/null || true

# Scripts (Python)
cp -r "$REPO/scripts/"* "$HA_CONFIG/scripts/" 2>/dev/null || true

# Node-RED Flows
cp "$REPO/nodered/flows.json" "/var/lib/homeassistant/addon_configs/a0d7b954_nodered/" 2>/dev/null || true

# Leere secrets.yaml erstellen (muss manuell ausgefüllt werden)
if [ ! -f "$HA_CONFIG/secrets.yaml" ]; then
    cp "$REPO/secrets.yaml.example" "$HA_CONFIG/secrets.yaml" 2>/dev/null || \
    echo "# secrets.yaml - Bitte alle Werte ausfüllen!" > "$HA_CONFIG/secrets.yaml"
fi

# Berechtigungen setzen
chown -R root:root "$HA_CONFIG"

Alle Konfigurationsdateien werden vom Repository in die Home Assistant Konfiguration kopiert. Die secrets.yaml wird als leere Vorlage erstellt und muss manuell mit den korrekten Werten ausgefüllt werden.

Phase 6: Docker Container starten

# Phase 6: Home Assistant Container
echo "=== Phase 6/7: Docker Container starten ==="
docker pull ghcr.io/home-assistant/qemux86-64-homeassistant:stable

docker run -d \
    --name homeassistant \
    --privileged \
    --restart=unless-stopped \
    -e TZ=Europe/Berlin \
    -v /var/lib/homeassistant/homeassistant:/config \
    -v /run/dbus:/run/dbus:ro \
    --network=host \
    ghcr.io/home-assistant/qemux86-64-homeassistant:stable

echo "Container gestartet. Warte auf Erstinitialisierung..."
sleep 30

# Prüfen ob Container läuft
if docker ps | grep -q homeassistant; then
    echo "Home Assistant Container läuft erfolgreich."
else
    echo "FEHLER: Container konnte nicht gestartet werden!"
    docker logs homeassistant
    exit 1
fi

Der Docker Container wird mit dem offiziellen HA OS Image gestartet. Die wichtigsten Parameter sind:

  • --privileged: Erlaubt Zugriff auf Hardware (USB, Bluetooth, etc.)
  • --network=host: Container nutzt das Host-Netzwerk direkt (für mDNS, MQTT etc.)
  • -v /var/lib/.../:/config: Mountet die Konfiguration in den Container
  • -e TZ=Europe/Berlin: Setzt die korrekte Zeitzone
  • --restart=unless-stopped: Automatischer Neustart bei Abstürzen

Phase 7: Abschluss und Statusbericht

# Phase 7: Zusammenfassung
echo "=== Phase 7/7: Abschluss ==="
echo ""
echo "================================================"
echo "  Wiederherstellung abgeschlossen!"
echo "================================================"
echo ""
echo "Home Assistant erreichbar unter:"
echo "  http://$(hostname -I | awk '{print $1}'):8123"
echo ""
echo "WICHTIG - Manuelle Nacharbeiten erforderlich:"
echo "  1. secrets.yaml ausfüllen: $HA_CONFIG/secrets.yaml"
echo "  2. HACS installieren und Custom Components nachinstallieren"
echo "  3. Alle Integrationen neu einrichten"
echo "  4. Node-RED Addon installieren"
echo "  5. Mosquitto Addon installieren"
echo ""
echo "Siehe Dokumentation: docs/15-wiederherstellung.html"
echo "================================================"

Manuelle Nacharbeiten

Nach Ausführung des restore.sh Scripts sind noch manuelle Schritte erforderlich, da bestimmte Daten aus Sicherheitsgründen nicht im Repository gespeichert werden.

1. secrets.yaml ausfüllen

Die wichtigste Nacharbeit ist das Ausfüllen der secrets.yaml mit den korrekten Werten. Ohne diese Datei können die meisten Integrationen nicht starten. 

# Datei bearbeiten:
sudo nano /var/lib/homeassistant/homeassistant/secrets.yaml

# Folgende Werte müssen eingetragen werden:
# - home_latitude / home_longitude / home_elevation
# - mqtt_user / mqtt_password
# - tibber_api_token
# - unifi_host / unifi_username / unifi_password
# - fritzbox_host / fritzbox_username / fritzbox_password
# - tractive_email / tractive_password
# - petkit_email / petkit_password
# - snmp_community
# - amazon_email / amazon_password
# - Alle Zonen-Koordinaten

# Nach dem Speichern HA neu starten:
docker restart homeassistant

2. HACS und Custom Components nachinstallieren

HACS (Home Assistant Community Store) muss manuell installiert werden, da es als Custom Component nicht über das Standard-Repository verteilt wird. 

# HACS installieren (im Container):
docker exec -it homeassistant bash
wget -O - https://get.hacs.xyz | bash -

# Alternativ von außen:
docker exec homeassistant bash -c \
  "wget -O - https://get.hacs.xyz | bash -"

# Container neu starten
docker restart homeassistant

Nach dem Neustart HACS in der UI einrichten und folgende Custom Components installieren:

Nr. Component Version Beschreibung
1 Alexa Media Player v5.11.0 Amazon Echo Steuerung
2 Tractive Aktuell GPS Haustier-Tracker
3 PetKit Aktuell Smarte Haustiergeräte
4 Dreame Vacuum Aktuell Saugroboter Willi/Robby
5 Fritz!Box Tools Aktuell Erweiterte Fritz!Box-Funktionen
6 UniFi Gateway Aktuell Erweiterte UniFi-Funktionen
7-12 Weitere Aktuell Lovelace-Karten, Theme-Plugins etc.

3. Integrationen neu einrichten

Alle Integrationen, die über die UI konfiguriert wurden, müssen neu eingerichtet werden, da deren Konfiguration im .storage-Verzeichnis lag, welches nicht im Git-Repository enthalten ist.

Reihenfolge der Einrichtung

  1. Mosquitto Addon installieren
    Einstellungen → Add-ons → Add-on Store → Mosquitto broker
    MQTT-Benutzer konfigurieren
  2. MQTT Integration
    Wird automatisch erkannt, wenn Mosquitto läuft
  3. Node-RED Addon installieren
    Einstellungen → Add-ons → Add-on Store → Node-RED
    Flows werden automatisch aus der kopierten flows.json geladen
  4. Tibber
    API-Token eingeben (von developer.tibber.com)
  5. Alexa Media Player
    Amazon-Login mit 2FA (kann beim ersten Versuch fehlschlagen - siehe Medien)
  6. Android TV (Sony Bravia)
    IP-Adresse eingeben, Pairing-Code auf dem TV bestätigen
  7. UniFi Network
    Controller-URL und Credentials eingeben
  8. Fritz!Box
    IP und Credentials eingeben
  9. Tractive und PetKit
    Jeweilige Cloud-Credentials eingeben
  10. DWD Wetterwarnungen, Dreame Vacuum etc.
    Weitere Integrationen nach Bedarf

4. Dashboards prüfen

YAML-basierte Dashboards sollten nach der Wiederherstellung automatisch funktionieren. UI-basierte Dashboards (aus .storage/lovelace) müssen ggf. manuell neu erstellt werden.

5. Automatisierungen testen

# YAML-Konfiguration prüfen
docker exec homeassistant python3 -m homeassistant --script check_config -c /config

# Oder über die UI:
# Entwicklerwerkzeuge > YAML > Konfiguration überprüfen

Wiederherstellung aus Backup

Falls ein vollständiges Backup (inkl. .storage und secrets.yaml) vorhanden ist, kann eine schnellere Wiederherstellung durchgeführt werden. 

# Backup entpacken
tar -xzf /mnt/backup/homeassistant/ha_backup_20260205_030000.tar.gz \
    -C /var/lib/homeassistant/

# Berechtigungen setzen
chown -R root:root /var/lib/homeassistant/homeassistant

# Container neu starten
docker restart homeassistant

# Vorteile:
# - secrets.yaml ist bereits ausgefüllt
# - .storage mit allen Integrationen ist vorhanden
# - Alle OAuth-Tokens sind noch gültig (falls nicht abgelaufen)
# - Dashboards (UI-Modus) sind wiederhergestellt
