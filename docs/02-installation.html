Installation

Diese Anleitung beschreibt die vollständige Ersteinrichtung des IceHomeAssist-Systems auf einem Intel NUC – von der Debian-Installation über Docker bis zum ersten Start von Home Assistant.

Voraussetzungen

Hardware-Anforderungen

Anforderung Minimum Empfohlen
Prozessor x86-64 kompatibel Intel NUC (i3/i5)
RAM 4 GB 8 GB oder mehr
Speicher 32 GB SSD 128 GB SSD oder mehr
Netzwerk Ethernet (LAN) Gigabit Ethernet

Software-Voraussetzungen

  • Debian 13 (Trixie) oder kompatible Distribution
  • Docker Engine (aktuelle stabile Version)
  • sudo-Zugang oder Root-Rechte
  • SSH-Zugang (empfohlen für Remote-Administration)
  • Stabile Internetverbindung für Cloud-Integrationen

Schritt 1: Debian installieren

Der Intel NUC wird mit einer frischen Debian-13-Installation eingerichtet. Verwende das offizielle Netinstall-Image für eine minimale Basisinstallation.

1.1 Installationsmedium erstellen

# Debian Netinstall ISO herunterladen
wget https://cdimage.debian.org/debian-cd/current/amd64/iso-cd/debian-13.*-amd64-netinst.iso

# USB-Stick erstellen (Achtung: /dev/sdX anpassen!)
sudo dd if=debian-13-amd64-netinst.iso of=/dev/sdX bs=4M status=progress
sync

1.2 Installationsoptionen

Empfohlene Auswahl während der Installation:
  • Sprache: Deutsch oder Englisch
  • Partitionierung: Geführt – gesamte Platte verwenden
  • Software-Auswahl: Nur SSH-Server und Standard-Systemwerkzeuge (keine Desktop-Umgebung)
  • GRUB-Bootloader installieren: Ja

1.3 System nach Installation einrichten

# System aktualisieren
sudo apt update && sudo apt upgrade -y

# Grundlegende Pakete installieren
sudo apt install -y \
    curl \
    wget \
    git \
    vim \
    htop \
    net-tools \
    ca-certificates \
    gnupg \
    lsb-release

# Statische IP-Adresse konfigurieren (empfohlen)
sudo vim /etc/network/interfaces

Beispiel: Statische IP-Konfiguration

# /etc/network/interfaces
auto enp0s31f6
iface enp0s31f6 inet static
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 192.168.1.1 8.8.8.8

Schritt 2: Docker installieren

Docker ist die Grundlage für den Home-Assistant-Container. Wir verwenden die offizielle Docker-Repository-Methode für die Installation.

2.1 Docker-Repository einrichten

# Alte Docker-Pakete entfernen (falls vorhanden)
sudo apt remove docker docker-engine docker.io containerd runc 2>/dev/null

# Docker GPG-Schlüssel hinzufügen
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Docker-Repository hinzufügen
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

2.2 Docker Engine installieren

# Paketindex aktualisieren
sudo apt update

# Docker Engine, CLI und Plugins installieren
sudo apt install -y \
    docker-ce \
    docker-ce-cli \
    containerd.io \
    docker-buildx-plugin \
    docker-compose-plugin

# Docker-Dienst aktivieren und starten
sudo systemctl enable docker
sudo systemctl start docker

# Benutzer zur Docker-Gruppe hinzufügen (optional)
sudo usermod -aG docker $USER

2.3 Installation verifizieren

# Docker-Version prüfen
docker --version
# Ausgabe: Docker version 27.x.x, build xxxxxxx

# Test-Container ausführen
sudo docker run hello-world
Achtung: Nach dem Hinzufügen zur Docker-Gruppe muss die Shell-Session neu gestartet werden (Ausloggen/Einloggen oder newgrp docker).

Schritt 3: Home Assistant Container einrichten

Das IceHomeAssist-System verwendet das offizielle HA OS Image für qemux86-64. Dieses Image enthält den vollständigen Home Assistant Supervisor mit Addon-Unterstützung.

3.1 Datenverzeichnis erstellen

# Verzeichnis für HA-Daten anlegen
sudo mkdir -p /var/lib/homeassistant/homeassistant
sudo mkdir -p /var/lib/homeassistant/addon_configs

3.2 Container starten

# Home Assistant Container starten
sudo docker run -d \
    --name homeassistant \
    --restart=unless-stopped \
    --privileged \
    --network=host \
    -v /var/lib/homeassistant:/data \
    -v /run/dbus:/run/dbus:ro \
    -e TZ="Europe/Berlin" \
    ghcr.io/home-assistant/qemux86-64-homeassistant
Pfad-Zuordnung: Das Host-Verzeichnis /var/lib/homeassistant/homeassistant entspricht innerhalb des Containers dem Pfad /config. Alle Konfigurationsdateien werden dort gespeichert.

3.3 Erste Einrichtung

  1. Warte ca. 2-5 Minuten, bis der Container vollständig gestartet ist
  2. Öffne im Browser: http://<NUC-IP>:8123
  3. Erstelle ein Administratorkonto
  4. Konfiguriere den Standort (für Wetter- und Sonnenstandsdaten)
  5. Wähle die Einheiten (metrisch)
  6. Überspringe die automatische Geräte-Erkennung zunächst

3.4 Container-Status prüfen

# Container-Status anzeigen
sudo docker ps | grep homeassistant

# Container-Logs anzeigen
sudo docker logs -f homeassistant --tail 50

# In den Container verbinden
sudo docker exec -it homeassistant bash

Nächste Schritte

Nach der erfolgreichen Ersteinrichtung folgen diese Schritte:

  1. Docker Setup – Detaillierte Container-Konfiguration und Management
  2. Konfiguration – configuration.yaml, Packages und Secrets einrichten
  3. Integrationen – HACS installieren und Custom Components einrichten
Wichtig: Alle Dateien im Konfigurationsverzeichnis gehören dem Benutzer root. Verwende immer sudo für Bearbeitungen auf dem Host oder nutze docker exec für Änderungen innerhalb des Containers.
