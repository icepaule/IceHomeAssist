Konfiguration

Die Konfiguration von Home Assistant erfolgt primär über YAML-Dateien. Diese Seite erklärt die Struktur der Hauptkonfiguration, das Package-System, das Secrets-Management und die allgemeine YAML-Syntax.

configuration.yaml

Die configuration.yaml ist die zentrale Konfigurationsdatei von Home Assistant. Sie bindet alle anderen Konfigurationsdateien ein und definiert grundlegende Einstellungen.

Grundstruktur

# configuration.yaml - Hauptkonfiguration

homeassistant:
  name: "IceHome"
  unit_system: metric
  time_zone: "Europe/Berlin"
  currency: EUR
  country: DE
  language: de
  packages: !include_dir_named packages

# Standard-Komponenten
default_config:

# Frontend-Konfiguration
frontend:
  themes: !include_dir_merge_named themes

# HTTP-Server
http:
  server_port: 8123
  ip_ban_enabled: true
  login_attempts_threshold: 5

# Logger-Konfiguration
logger:
  default: warning
  logs:
    homeassistant.components.mqtt: info
    custom_components.alexa_media: debug

# Recorder-Konfiguration (Datenbank)
recorder:
  purge_keep_days: 14
  commit_interval: 1

# Automatisierungen einbinden
automation: !include automations.yaml
script: !include scripts.yaml
scene: !include scenes.yaml

# Lovelace Dashboards
lovelace:
  mode: yaml
  resources: []
  dashboards:
    home-dashboard:
      mode: yaml
      filename: dashboards/home.yaml
      title: Home
    # ... weitere Dashboards (siehe Dashboards-Seite)

Packages-System

Das Package-System ist eine der leistungsfähigsten Organisationsmethoden in Home Assistant. Anstatt alle Sensoren, Automatisierungen und Scripts in einer einzigen Datei zu definieren, können zusammengehörige Konfigurationen in einzelne Dateien aufgeteilt werden.

Prinzip: Jedes Package ist eine eigenständige YAML-Datei, die alle Konfigurationseinträge für einen bestimmten Anwendungsfall enthält – Sensoren, Automatisierungen, Scripts, Input-Helpers und mehr.

Package-Einbindung

# In configuration.yaml:
homeassistant:
  packages: !include_dir_named packages

# Dies lädt automatisch alle .yaml Dateien aus dem packages/ Verzeichnis.
# Der Dateiname (ohne .yaml) wird als Package-Name verwendet.

Beispiel: Package-Struktur

# packages/energy.yaml - Beispiel für ein Package

# Sensoren
sensor:
  - platform: mqtt
    name: "PV Leistung"
    state_topic: "tele/pv_sensor/SENSOR"
    value_template: "{{ value_json.ENERGY.Power }}"
    unit_of_measurement: "W"
    device_class: power

# Template-Sensoren
template:
  - sensor:
    - name: "Gesamtverbrauch"
      state: "{{ states('sensor.server_power')|float + states('sensor.kueche_power')|float }}"
      unit_of_measurement: "W"

# Input-Helpers
input_number:
  energy_daily_target:
    name: "Tägliches Energieziel"
    min: 0
    max: 50
    step: 0.5
    unit_of_measurement: "kWh"

Secrets-Management

Sensible Daten wie Passwörter, API-Schlüssel und Tokens werden in der Datei secrets.yaml gespeichert und in der Konfiguration über !secret referenziert.

secrets.yaml Aufbau

# secrets.yaml - NIEMALS in Git committen!

# MQTT
mqtt_username: "ha_mqtt_user"
mqtt_password: "***GEHEIM***"

# Tibber
tibber_token: "***GEHEIM***"

# Fritz!Box
fritzbox_password: "***GEHEIM***"

# Divera
divera_api_key: "***GEHEIM***"

# ... weitere Secrets

Verwendung in der Konfiguration

# Referenz auf ein Secret
mqtt:
  broker: localhost
  port: 1883
  username: !secret mqtt_username
  password: !secret mqtt_password
Sicherheitshinweis: Die Datei secrets.yaml sollte niemals in ein Git-Repository committed werden. Stelle sicher, dass sie in der .gitignore aufgeführt ist. Siehe Sicherheit für weitere Details.

YAML-Syntax Grundlagen

Wichtige YAML-Regeln

Konzept Syntax Beispiel
Schlüssel-Wert key: value name: "Mein Sensor"
Verschachtelung 2 Leerzeichen Einrückung   child: value
Listen - item - platform: mqtt
Kommentare # Kommentar # Dies ist ein Kommentar
Strings Anführungszeichen (optional) "Text mit Sonderzeichen"
Multiline | oder > description: |
Include !include datei.yaml automation: !include automations.yaml
Secret !secret key password: !secret mqtt_password

Häufige YAML-Fehler

Typische Fehlerquellen:
  • Tabs statt Leerzeichen: YAML erlaubt keine Tabs zur Einrückung – verwende immer Leerzeichen
  • Falsche Einrückung: Jede Ebene muss exakt 2 Leerzeichen eingerückt sein
  • Fehlende Anführungszeichen: Strings mit Sonderzeichen (:, #, {) müssen in Anführungszeichen stehen
  • Doppelte Schlüssel: Jeder Schlüssel darf auf einer Ebene nur einmal vorkommen

Konfiguration validieren

# Konfiguration innerhalb des Containers prüfen
sudo docker exec homeassistant ha core check

# Oder über die HA Web-Oberfläche:
# Einstellungen → System → "Konfiguration prüfen"
Best Practice: Prüfe die Konfiguration immer vor einem Neustart von Home Assistant. Fehlerhafte YAML-Dateien können verhindern, dass HA startet.
