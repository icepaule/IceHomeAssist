Medien

Die Mediensteuerung umfasst Amazon Alexa Geräte über die HACS-Integration Alexa Media Player sowie den Sony Bravia Fernseher über die Android TV Integration. Zusammen ermöglichen sie eine zentrale Steuerung aller Mediengeräte im Haus, einschließlich Text-to-Speech Durchsagen und Multiroom-Audio.

Das Medien Dashboard mit Steuerung aller Mediengeräte

Alexa Media Player (HACS)

Die Alexa Media Player Integration (Version 5.11.0) ist eine HACS Custom Component, die Amazon Echo Geräte in Home Assistant einbindet. Sie ermöglicht die vollständige Steuerung aller Echo-Funktionen: Medienwiedergabe, Lautstärke, Text-to-Speech, Routinen und mehr.

Installierte Geräte

Entity Gerät Standort Funktionen
media_player.marcus_echo_show_8_2nd_gen Echo Show 8 (2. Gen.) Wohnzimmer Audio, Video, Display, TTS
media_player.marcus_echo_dot Echo Dot Bad Audio, TTS
media_player.marcus_echo_plus Echo Plus Küche Audio, TTS
media_player.uberall Gruppe "Überall" Alle Räume Multiroom-Audio, TTS

Einrichtung der Integration

Die Einrichtung der Alexa Media Player Integration erfordert besondere Aufmerksamkeit, da Amazon strenge Bot-Erkennungsmechanismen einsetzt.

Voraussetzungen

  • HACS muss installiert und eingerichtet sein
  • Amazon-Konto mit aktivierter Zwei-Faktor-Authentifizierung (2FA)
  • Die 2FA ist zwingend erforderlich, um Amazons Bot-Erkennung zu umgehen

Installationsschritte

  1. In HACS nach Alexa Media Player suchen und installieren
  2. Home Assistant neu starten
  3. Einstellungen → Geräte & Dienste → Integration hinzufügen
  4. Nach Alexa Media Player suchen
  5. Amazon-E-Mail und Passwort eingeben
  6. Den 2FA-Code eingeben, wenn aufgefordert
  7. Bei Erfolg erscheinen die Echo-Geräte automatisch als Entities

Häufiges Problem: HTTP 500 Fehler

Beim Setup kann es zu 500 Internal Server Error kommen. Dies ist kein Fehler in Home Assistant oder der Integration, sondern Amazons Bot-Erkennung (intern "aamation" Challenge genannt). Lösungsansätze:

  • 2FA auf dem Amazon-Konto aktivieren (wichtigster Schritt)
  • Einige Minuten warten und erneut versuchen
  • VPN deaktivieren, falls aktiv
  • Einen anderen Browser für den Amazon-Login verwenden
  • Das Amazon-Konto vorher im Browser auf demselben Gerät einloggen

Konfigurationsoptionen

# Die Alexa Media Player Integration wird über die UI konfiguriert.
# Folgende Optionen sind verfügbar:

# E-Mail:          Amazon Login E-Mail
# Passwort:        Amazon Login Passwort
# 2FA Code:        Aktueller 2FA-Code
# Amazon Domain:   amazon.de (für Deutschland)
# Include Devices: Komma-getrennte Liste (leer = alle)
# Exclude Devices: Geräte ausschließen
# Scan Interval:   Aktualisierungsintervall (Standard: 60s)

Text-to-Speech (TTS) über Alexa

Eine der nützlichsten Funktionen ist die Möglichkeit, Sprachdurchsagen über die Echo-Geräte zu machen. Dies wird für Benachrichtigungen aus Automatisierungen verwendet, z.B. wenn die Waschmaschine fertig ist oder der Strompreis sich ändert.

TTS-Beispiele

# Einzelnes Gerät ansprechen
service: notify.alexa_media_marcus_echo_show_8_2nd_gen
data:
  message: "Die Waschmaschine ist fertig."
  data:
    type: tts

# Alle Geräte gleichzeitig (Multiroom)
service: notify.alexa_media_uberall
data:
  message: "Achtung, der Strompreis ist jetzt sehr günstig!"
  data:
    type: tts

# TTS mit Lautstärke-Anpassung
service: notify.alexa_media_marcus_echo_show_8_2nd_gen
data:
  message: "Guten Morgen! Es ist {{ now().strftime('%H:%M') }} Uhr."
  data:
    type: tts

# Ankündigung (announce) - unterbricht laufende Wiedergabe
service: notify.alexa_media_marcus_echo_show_8_2nd_gen
data:
  message: "Wichtige Durchsage: Die Haustür wurde geöffnet."
  data:
    type: announce

TTS in Automatisierungen

# Beispiel: Täglicher Wetterbericht
- alias: "Morgen-Briefing"
  trigger:
    - platform: time
      at: "07:00:00"
  condition:
    - condition: state
      entity_id: person.marcus
      state: "home"
  action:
    - service: notify.alexa_media_marcus_echo_show_8_2nd_gen
      data:
        message: >
          Guten Morgen! Heute ist {{ now().strftime('%A, der %d. %B') }}.
          Die aktuelle Außentemperatur beträgt
          {{ states('sensor.openweathermap_temperature') }} Grad.
          Der Strompreis liegt bei
          {{ states('sensor.electricity_price') }} Cent pro Kilowattstunde.
        data:
          type: tts

Song-Info mit Erscheinungsjahr (Hitster-Training)

Beim Radiohören über Alexa (TuneIn) werden automatisch Interpret, Titel und Erscheinungsjahr des aktuellen Songs ermittelt und angezeigt. Die Jahreszahl wird über die MusicBrainz API (kostenlos, kein API-Key) abgefragt. Das Feature dient als „Hitster-Training“ – man kann raten, aus welchem Jahr ein Song stammt.

Funktionsweise

Ein Node-RED Flow überwacht alle drei Alexa-Geräte auf Song-Wechsel. Alexa liefert bei TuneIn-Radiosendern den Song im Format media_title = "Artist: Titel" und den Sendernamen in media_artist. Der Flow:

  1. Erkennt Song-Wechsel und filtert Sendernamen, Werbung, Duplikate
  2. Fragt die MusicBrainz API nach dem Erscheinungsjahr
  3. Publiziert das Ergebnis per MQTT (Topic: song-info/current)
  4. Zeigt optional eine Announce auf aktiven Alexas (Text + Sprache)

Steuerung per Toggle

Jede Alexa hat einen eigenen input_boolean Toggle im Dashboard. Ist der Toggle aktiv, erhält dieses Gerät die Song-Announce. MQTT läuft immer.

Toggle Alexa Standard
input_boolean.song_info_echo_show Echo Show 8 (Wohnzimmer) An
input_boolean.song_info_echo_dot Echo Dot Aus
input_boolean.song_info_echo_plus Echo Plus Aus

HA Package

# packages/song_info.yaml

mqtt:
  sensor:
    - name: "Aktueller Song"
      unique_id: aktueller_song
      state_topic: "song-info/current"
      value_template: "{{ value_json.artist }} - {{ value_json.title }} ({{ value_json.year }})"
      json_attributes_topic: "song-info/current"
      icon: mdi:music-note

    - name: "Aktueller Sender"
      unique_id: aktueller_sender
      state_topic: "song-info/current"
      value_template: "{{ value_json.source }}"
      icon: mdi:radio

input_boolean:
  song_info_echo_show:
    name: "Song-Info Echo Show"
    initial: true
  song_info_echo_dot:
    name: "Song-Info Echo Dot"
    initial: false
  song_info_echo_plus:
    name: "Song-Info Echo Plus"
    initial: false

Announce-Methoden

Methode Verhalten
method: "all" Text anzeigen + Sprachausgabe, Musik wird kurz stummgeschaltet
method: "show" Nur Text-Anzeige (lautlos), Musik läuft weiter
method: "speak" Nur Sprachausgabe, kein Text
Einschränkung: Song-Metadaten werden nur von TuneIn geliefert (z.B. „Alexa, spiel Bayern 1“). Die ARD Audiothek liefert keine Titelinformationen über die Alexa-API.

Android TV - Sony Bravia

Der Sony Bravia Fernseher ist über die Android TV Integration in Home Assistant eingebunden. Die Integration ermöglicht das Ein-/Ausschalten, die App-Steuerung und die Anzeige des aktuellen Wiedergabestatus.

Integration einrichten

  1. Einstellungen → Geräte & Dienste → Integration hinzufügen
  2. Nach Android TV suchen
  3. Die IP-Adresse des Fernsehers eingeben
  4. Den Gerätetyp Android TV auswählen
  5. Den auf dem Fernseher angezeigten Pairing-Code eingeben
  6. Die ADB-Verbindung wird hergestellt und das Gerät erscheint

Verfügbare Entity

Entity Typ Beschreibung
media_player.sony_bravia Media Player Hauptsteuerung des Fernsehers
remote.sony_bravia Remote Fernbedienungsfunktionen

Steuerungsmöglichkeiten

# Fernseher einschalten
service: media_player.turn_on
target:
  entity_id: media_player.sony_bravia

# Fernseher ausschalten
service: media_player.turn_off
target:
  entity_id: media_player.sony_bravia

# App starten (z.B. Netflix)
service: media_player.select_source
target:
  entity_id: media_player.sony_bravia
data:
  source: "Netflix"

# Lautstärke regeln
service: media_player.volume_set
target:
  entity_id: media_player.sony_bravia
data:
  volume_level: 0.2

# Taste senden (über Remote Entity)
service: remote.send_command
target:
  entity_id: remote.sony_bravia
data:
  command: "HOME"

Automatisierungen mit dem Fernseher

# Fernseher automatisch nach Inaktivität ausschalten
- alias: "TV Auto-Off nach Inaktivität"
  trigger:
    - platform: state
      entity_id: media_player.sony_bravia
      to: "idle"
      for:
        hours: 2
  condition:
    - condition: time
      after: "23:00:00"
      before: "06:00:00"
  action:
    - service: media_player.turn_off
      target:
        entity_id: media_player.sony_bravia
    - service: notify.alexa_media_marcus_echo_show_8_2nd_gen
      data:
        message: "Der Fernseher wurde nach zwei Stunden Inaktivität ausgeschaltet."
        data:
          type: tts

Medien Dashboard

Das Medien Dashboard bietet eine zentrale Übersicht und Steuerung aller Mediengeräte. Es verwendet verschiedene Lovelace-Karten.

Dashboard-Aufbau

# dashboards/medien.yaml
views:
  - title: Medien
    icon: mdi:multimedia
    cards:
      # Alexa Echo Show - Steuerung
      - type: media-control
        entity: media_player.marcus_echo_show_8_2nd_gen

      # Alexa Gruppe - Überall
      - type: media-control
        entity: media_player.uberall

      # Sony Bravia TV
      - type: media-control
        entity: media_player.sony_bravia

      # Schnellzugriff: TTS Durchsage
      - type: entities
        title: Schnellaktionen
        entities:
          - type: button
            name: "Test-Durchsage"
            tap_action:
              action: call-service
              service: notify.alexa_media_marcus_echo_show_8_2nd_gen
              data:
                message: "Dies ist eine Test-Durchsage."
                data:
                  type: tts

      # Geräte-Status
      - type: entities
        title: Medienstatus
        entities:
          - entity: media_player.marcus_echo_show_8_2nd_gen
            name: Echo Show 8
          - entity: media_player.uberall
            name: Alexa Überall
          - entity: media_player.sony_bravia
            name: Sony Bravia TV

Alexa Routinen und Skills

Über die Alexa Media Player Integration können auch Alexa-Routinen ausgelöst und benutzerdefinierte Befehle gesendet werden.

Alexa-Routine auslösen

# Routine über Sequenz-Befehl auslösen
service: media_player.play_media
target:
  entity_id: media_player.marcus_echo_show_8_2nd_gen
data:
  media_content_id: "Guten Morgen Routine"
  media_content_type: "routine"

Alexa-Befehle senden

# Sprachbefehl simulieren
service: media_player.play_media
target:
  entity_id: media_player.marcus_echo_show_8_2nd_gen
data:
  media_content_id: "Wie wird das Wetter heute?"
  media_content_type: "custom"

Tipps und Best Practices

  • TTS-Lautstärke beachten: Nachts sollte die TTS-Lautstärke reduziert oder komplett deaktiviert werden. Dies kann über eine Zeitbedingung in Automatisierungen gesteuert werden.
  • Gruppe nutzen: Für wichtige Durchsagen die Gruppe "Überall" verwenden, damit die Nachricht in jedem Raum gehört wird.
  • 2FA aktuell halten: Die Alexa-Integration muss gelegentlich neu authentifiziert werden. 2FA-Codes bereithalten.
  • Netzwerkstabilität: Alle Mediengeräte sollten eine stabile WLAN- oder LAN-Verbindung haben, um Ausfälle zu vermeiden.
  • TV-Quellen benennen: Die Android TV Quellen (Apps) können in Home Assistant mit benutzerdefinierten Namen versehen werden für eine bessere Übersicht.
  • Medien-Timeout: Bei der Alexa-Integration kann es zu Timeouts kommen, wenn das Amazon-Cookie abgelaufen ist. In diesem Fall die Integration neu konfigurieren.
