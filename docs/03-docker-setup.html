Docker Setup

Das IceHomeAssist-System läuft als Docker-Container auf einem Intel NUC. Diese Seite dokumentiert die vollständige Docker-Konfiguration, einschließlich Image, Volumes, Netzwerk und Container-Management.

Container Image

Image-Details

Eigenschaft Wert
Image ghcr.io/home-assistant/qemux86-64-homeassistant
Registry GitHub Container Registry (ghcr.io)
Architektur qemux86-64 (x86-64 Emulation)
Typ HA OS (mit Supervisor und Addon-Unterstützung)
Container-Name homeassistant
Warum qemux86-64? Dieses Image ist speziell für x86-64-Systeme wie den Intel NUC optimiert. Es enthält den vollständigen HA OS Stack einschließlich Supervisor, der die Verwaltung von Addons wie Node-RED und Mosquitto ermöglicht.

Docker Run Befehl

Der folgende Befehl startet den Home-Assistant-Container mit allen erforderlichen Parametern: 

sudo docker run -d \
    --name homeassistant \
    --restart=unless-stopped \
    --privileged \
    --network=host \
    -v /var/lib/homeassistant:/data \
    -v /run/dbus:/run/dbus:ro \
    -e TZ="Europe/Berlin" \
    ghcr.io/home-assistant/qemux86-64-homeassistant

Parameter-Erklärung

Parameter Beschreibung
-d Startet den Container im Hintergrund (detached mode)
--name homeassistant Vergibt den Container-Namen homeassistant für einfache Referenzierung
--restart=unless-stopped Automatischer Neustart bei Absturz oder Systemstart (außer manuell gestoppt)
--privileged Voller Zugriff auf Host-Geräte (erforderlich für Hardware-Zugriff, USB, Bluetooth)
--network=host Container nutzt den Netzwerk-Stack des Hosts (erforderlich für mDNS, SSDP, MQTT Discovery)
-v /var/lib/homeassistant:/data Persistenter Speicher: Host-Verzeichnis wird in den Container gemountet
-v /run/dbus:/run/dbus:ro D-Bus-Zugriff (read-only) für Bluetooth und Systemdienste
-e TZ="Europe/Berlin" Setzt die Zeitzone auf Mitteleuropäische Zeit

Volume Mounts

Verzeichnisstruktur auf dem Host

/var/lib/homeassistant/           # Hauptverzeichnis (gemountet als /data)
├── homeassistant/               # HA-Konfiguration (= /config im Container)
│   ├── configuration.yaml       # Hauptkonfigurationsdatei
│   ├── secrets.yaml             # Passwörter und API-Keys
│   ├── automations.yaml         # Automatisierungen
│   ├── packages/                # YAML Packages
│   ├── dashboards/              # Dashboard YAML-Dateien
│   ├── custom_components/       # HACS Integrationen
│   ├── themes/                  # Custom Themes
│   ├── www/                     # Statische Dateien (Bilder, Icons)
│   └── .storage/                # HA-interne Datenbank
├── addon_configs/               # Addon-Konfigurationen
│   └── a0d7b954_nodered/        # Node-RED Konfiguration
│       └── flows.json           # Node-RED Flows
├── addons/                      # Installierte Addons
├── backup/                      # Automatische Backups
├── media/                       # Mediendateien
├── share/                       # Geteilte Daten zwischen Addons
└── ssl/                         # SSL-Zertifikate
Dateiberechtigungen: Alle Dateien im Konfigurationsverzeichnis gehören dem Benutzer root. Bearbeitungen auf dem Host erfordern sudo. Alternativ kann docker exec -it homeassistant bash verwendet werden, um Dateien innerhalb des Containers zu bearbeiten.

Pfad-Zuordnung

Host-Pfad Container-Pfad Beschreibung
/var/lib/homeassistant /data Hauptdaten-Verzeichnis
/var/lib/homeassistant/homeassistant /config HA-Konfigurationsverzeichnis
/var/lib/homeassistant/addon_configs/a0d7b954_nodered (Addon-intern) Node-RED Konfiguration und Flows
/run/dbus /run/dbus (ro) D-Bus Socket (read-only)

Netzwerk-Konfiguration

Host-Netzwerk-Modus

Der Container läuft im --network=host-Modus. Das bedeutet:

  • Der Container teilt sich den Netzwerk-Stack mit dem Host
  • Alle Ports des Containers sind direkt auf dem Host verfügbar
  • Kein Port-Mapping erforderlich (-p wird nicht benötigt)
  • mDNS-Discovery funktioniert korrekt (z.B. für Chromecast, ESPHome)
  • MQTT-Broadcasts und Multicasts werden durchgeleitet

Verwendete Ports

Port Dienst Beschreibung
8123 Home Assistant Web-Interface und API
1880 Node-RED Flow-Editor
1883 Mosquitto MQTT MQTT Broker (unverschlüsselt)
5580 Matter Server Matter-Protokoll
8884 Mosquitto MQTT MQTT über WebSocket (optional)

Container-Management

Grundlegende Befehle

# Container-Status anzeigen
sudo docker ps | grep homeassistant

# Container stoppen
sudo docker stop homeassistant

# Container starten
sudo docker start homeassistant

# Container neu starten
sudo docker restart homeassistant

# Container-Logs anzeigen (letzte 100 Zeilen, live)
sudo docker logs -f homeassistant --tail 100

# In den Container verbinden (Shell)
sudo docker exec -it homeassistant bash

# Befehl im Container ausführen
sudo docker exec homeassistant ha core check

Container aktualisieren

# Neues Image herunterladen
sudo docker pull ghcr.io/home-assistant/qemux86-64-homeassistant:latest

# Alten Container stoppen und entfernen
sudo docker stop homeassistant
sudo docker rm homeassistant

# Neuen Container mit gleichen Parametern starten
sudo docker run -d \
    --name homeassistant \
    --restart=unless-stopped \
    --privileged \
    --network=host \
    -v /var/lib/homeassistant:/data \
    -v /run/dbus:/run/dbus:ro \
    -e TZ="Europe/Berlin" \
    ghcr.io/home-assistant/qemux86-64-homeassistant:latest
Achtung: Vor einem Update immer ein Backup erstellen! Änderungen an Python-Bibliotheken innerhalb des Containers (z.B. unter /usr/local/lib/python3.13/...) gehen bei einem Container-Neustart oder Update verloren. Siehe Wiederherstellung.

Ressourcen-Überwachung

# CPU und Speicherverbrauch des Containers
sudo docker stats homeassistant --no-stream

# Detaillierte Container-Informationen
sudo docker inspect homeassistant

# Disk-Usage aller Docker-Ressourcen
sudo docker system df

Aufräumen

# Nicht mehr verwendete Images entfernen
sudo docker image prune -a

# Alle nicht verwendeten Docker-Ressourcen bereinigen
sudo docker system prune
Tipp: Regelmäßiges Aufräumen alter Images spart Festplattenspeicher. Nach einem Update können alte Image-Versionen mehrere Gigabyte belegen.
