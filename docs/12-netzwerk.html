Netzwerk

Das Netzwerk-Monitoring umfasst die UniFi-Infrastruktur mit Radar-Visualisierung verbundener Geräte, MikroTik Managed Switches mit SNMP-Überwachung sowie die Fritz!Box als Internet-Gateway. Über eigene Python-Skripte werden die Daten aufbereitet und als Sensoren in Home Assistant bereitgestellt.

UniFi Netzwerk

Die WLAN-Infrastruktur basiert auf Ubiquiti UniFi Access Points. Die UniFi-Integration in Home Assistant liefert Informationen über verbundene Clients, Signalstärken und Netzwerkauslastung. Zusätzlich wird ein eigenes Python-Script für eine Radar-Visualisierung eingesetzt.

UniFi Integration einrichten

  1. Einstellungen → Geräte & Dienste → Integration hinzufügen
  2. Nach UniFi Network suchen
  3. Host: IP-Adresse des UniFi Controllers
  4. Benutzername: Lokaler UniFi Admin-Account
  5. Passwort: Aus secrets.yaml
  6. Port: 443 (Standard für UniFi OS)
  7. SSL-Zertifikat verifizieren: Nein (bei selbstsigniertem Zertifikat)

Verfügbare Entities

Entity-Typ Beschreibung Beispiel
device_tracker Präsenzerkennung pro WLAN-Client device_tracker.marcus_handy
sensor Uptime, TX/RX Datenraten sensor.unifi_ap_clients
switch Port-Blockierung, PoE-Steuerung switch.unifi_poe_port_1
update Firmware-Updates für UniFi-Geräte update.unifi_ap_firmware

Radar-Visualisierung (unifi_radar_data.py)

Ein eigenes Python-Script generiert eine SVG-basierte Radar-Darstellung aller mit dem UniFi-Netzwerk verbundenen und in der Nähe befindlichen Clients. Das Radar zeigt die Signalstärke als Entfernung vom Zentrum und unterscheidet zwischen verbundenen (Connected) und lediglich sichtbaren (Nearby) Geräten.

Funktionsweise

  1. Das Script verbindet sich mit der UniFi Controller API
  2. Es fragt die aktuelle Client-Liste und Nearby-Devices ab
  3. Für jeden Client wird die Signalstärke (RSSI) ausgelesen
  4. Die Daten werden in eine SVG-Radar-Grafik umgewandelt
  5. Die SVG-Datei wird als lokale Datei gespeichert und in Home Assistant angezeigt

Script-Konfiguration

# unifi_radar_data.py - Konfiguration
# Dieses Script wird als shell_command oder über einen Cron-Job ausgeführt

UNIFI_HOST = "https://192.168.1.1"     # UniFi Controller URL
UNIFI_USER = "admin"                    # UniFi Benutzername
UNIFI_PASS = ""                         # Aus Environment-Variable
UNIFI_SITE = "default"                  # Site-Name

OUTPUT_FILE = "/config/www/unifi_radar.svg"  # Ausgabedatei

# Radar-Parameter
RADAR_RINGS = 4                         # Anzahl konzentrischer Ringe
RADAR_SIZE = 400                        # SVG-Größe in Pixel
CONNECTED_COLOR = "#4CAF50"             # Grün für verbundene Clients
NEARBY_COLOR = "#FF9800"                # Orange für Nearby-Clients
BACKGROUND_COLOR = "#1a1a2e"            # Dunkler Hintergrund

SVG-Radar Ausgabe

# Das generierte SVG zeigt:
# - Konzentrische Ringe: Signalstärke (-30dBm bis -90dBm)
# - Grüne Punkte: Verbundene Clients (Connected)
# - Orange Punkte: Sichtbare Geräte in der Nähe (Nearby)
# - Labels: Gerätename und Signalstärke

# Einbindung im Dashboard:

MikroTik Switches

Im Netzwerk werden drei MikroTik Managed Switches eingesetzt, die über SNMP (Simple Network Management Protocol) überwacht werden. Die Switches bilden das kabelgebundene Backbone des Heimnetzwerks.

Switch-Übersicht

Modell Standort Ports Besonderheiten
CSS326-24G-2S+ Keller (Hauptverteiler) 24x 1GbE + 2x SFP+ Zentraler Switch, PoE-fähig
CSS326-24G-2S+ Arbeitszimmer 24x 1GbE + 2x SFP+ Desktop-Switch für Arbeitsplatz
CRS305-1G-4S+ Keller (10G-Backbone) 1x 1GbE + 4x SFP+ 10 Gbit/s Uplinks zwischen Switches

SNMP-Konfiguration auf MikroTik

SNMP muss auf jedem MikroTik Switch aktiviert werden. Dies erfolgt über die MikroTik Web-Oberfläche (WebFig) oder per SSH: 

# MikroTik RouterOS - SNMP aktivieren
/snmp set enabled=yes contact="admin" location="Keller"
/snmp community set 0 name=public read-access=yes write-access=no

# SNMP-Einstellungen prüfen
/snmp print
/snmp community print

SNMP Monitoring Script (snmp_switch_ports.py)

Ein eigenes Python-Script liest über SNMP die Port-Statistiken aller MikroTik Switches aus und stellt sie als Sensoren in Home Assistant bereit. Das Script ermittelt pro Port: Link-Status, Geschwindigkeit, TX/RX Bytes und Fehlerrate. 

# snmp_switch_ports.py - Auszug
import subprocess
import json

# SNMP OIDs für Interface-Monitoring
OIDS = {
    "ifDescr":      "1.3.6.1.2.1.2.2.1.2",     # Port-Beschreibung
    "ifOperStatus":  "1.3.6.1.2.1.2.2.1.8",     # Betriebsstatus (1=up, 2=down)
    "ifSpeed":       "1.3.6.1.2.1.2.2.1.5",     # Geschwindigkeit
    "ifInOctets":    "1.3.6.1.2.1.2.2.1.10",    # Empfangene Bytes
    "ifOutOctets":   "1.3.6.1.2.1.2.2.1.16",    # Gesendete Bytes
    "ifInErrors":    "1.3.6.1.2.1.2.2.1.14",    # Empfangsfehler
    "ifOutErrors":   "1.3.6.1.2.1.2.2.1.20",    # Sendefehler
}

SWITCHES = {
    "keller":        {"host": "192.168.1.10", "community": "public"},
    "arbeitszimmer": {"host": "192.168.1.11", "community": "public"},
    "backbone":      {"host": "192.168.1.12", "community": "public"},
}

def snmp_walk(host, community, oid):
    """Führt einen SNMP Walk aus und gibt die Ergebnisse zurück."""
    cmd = f"snmpwalk -v2c -c {community} {host} {oid}"
    result = subprocess.run(cmd.split(), capture_output=True, text=True)
    return parse_snmp_output(result.stdout)

def get_port_status(host, community):
    """Ermittelt den Status aller Ports eines Switches."""
    ports = {}
    descriptions = snmp_walk(host, community, OIDS["ifDescr"])
    statuses = snmp_walk(host, community, OIDS["ifOperStatus"])
    speeds = snmp_walk(host, community, OIDS["ifSpeed"])

    for idx in descriptions:
        ports[idx] = {
            "name": descriptions[idx],
            "status": "up" if statuses.get(idx) == "1" else "down",
            "speed": int(speeds.get(idx, 0)) // 1000000,  # Mbps
        }
    return ports

# Ergebnis als JSON für Home Assistant REST-Sensor
if __name__ == "__main__":
    result = {}
    for name, config in SWITCHES.items():
        result[name] = get_port_status(config["host"], config["community"])
    print(json.dumps(result, indent=2))

Fritz!Box Integration

Die Fritz!Box dient als Internet-Gateway und wird über die native Home Assistant Integration überwacht. Sie liefert Informationen über die Internetverbindung, verbundene Geräte und den DSL-Status.

Integration einrichten

  1. Einstellungen → Geräte & Dienste → Integration hinzufügen
  2. Nach AVM FRITZ!Box suchen
  3. Host: IP-Adresse der Fritz!Box (Standard: 192.168.178.1)
  4. Benutzername: Fritz!Box Admin-Benutzer
  5. Passwort: Fritz!Box Passwort aus secrets.yaml

Verfügbare Sensoren

Entity Beschreibung
sensor.fritz_box_connection_uptime Verbindungsdauer seit letztem Reconnect
sensor.fritz_box_download_throughput Aktuelle Download-Geschwindigkeit
sensor.fritz_box_upload_throughput Aktuelle Upload-Geschwindigkeit
sensor.fritz_box_max_download Maximale Download-Rate (Sync)
sensor.fritz_box_max_upload Maximale Upload-Rate (Sync)
sensor.fritz_box_external_ip Aktuelle externe IP-Adresse
binary_sensor.fritz_box_connection Internetverbindung aktiv (ja/nein)

Netzwerk-Monitoring Automatisierungen

# Benachrichtigung bei Internetausfall
- alias: "Internet Ausfall Warnung"
  trigger:
    - platform: state
      entity_id: binary_sensor.fritz_box_connection
      to: "off"
      for:
        minutes: 2
  action:
    - service: persistent_notification.create
      data:
        title: "Internetausfall"
        message: >
          Die Internetverbindung ist seit 2 Minuten unterbrochen.
          Letzte bekannte externe IP: {{ states('sensor.fritz_box_external_ip') }}

# Benachrichtigung bei Switch-Ausfall
- alias: "Switch Offline Warnung"
  trigger:
    - platform: state
      entity_id:
        - binary_sensor.switch_keller_online
        - binary_sensor.switch_az_online
        - binary_sensor.switch_backbone_online
      to: "off"
      for:
        minutes: 5
  action:
    - service: notify.alexa_media_marcus_echo_show_8_2nd_gen
      data:
        message: >
          Achtung! Der Netzwerk-Switch {{ trigger.to_state.attributes.friendly_name }}
          ist seit 5 Minuten nicht erreichbar.
        data:
          type: tts
