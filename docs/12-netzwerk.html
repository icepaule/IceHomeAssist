Netzwerk

Das Netzwerk-Monitoring umfasst die UniFi-Infrastruktur mit Radar-Visualisierung verbundener Geräte, MikroTik und Cisco Managed Switches mit SNMP-Überwachung, die Fritz!Box als Internet-Gateway sowie eine automatisch generierte Network Admin Page, die alle Netzwerkgeräte (aus AdGuard-DNS-Daten) mit klickbaren Web/SSH/Telnet-Links anzeigt. Über eigene Python-Skripte werden die Daten aufbereitet und als Sensoren in Home Assistant bereitgestellt.

Das Netzwerk Dashboard mit Switch-Status, WLAN-Clients und Internetüberwachung

UniFi Netzwerk

Die WLAN-Infrastruktur basiert auf Ubiquiti UniFi Access Points. Die UniFi-Integration in Home Assistant liefert Informationen über verbundene Clients, Signalstärken und Netzwerkauslastung. Zusätzlich wird ein eigenes Python-Script für eine Radar-Visualisierung eingesetzt.

UniFi Integration einrichten

  1. Einstellungen → Geräte & Dienste → Integration hinzufügen
  2. Nach UniFi Network suchen
  3. Host: IP-Adresse des UniFi Controllers
  4. Benutzername: Lokaler UniFi Admin-Account
  5. Passwort: Aus secrets.yaml
  6. Port: 443 (Standard für UniFi OS)
  7. SSL-Zertifikat verifizieren: Nein (bei selbstsigniertem Zertifikat)

Verfügbare Entities

Entity-Typ Beschreibung Beispiel
device_tracker Präsenzerkennung pro WLAN-Client device_tracker.marcus_handy
sensor Uptime, TX/RX Datenraten sensor.unifi_ap_clients
switch Port-Blockierung, PoE-Steuerung switch.unifi_poe_port_1
update Firmware-Updates für UniFi-Geräte update.unifi_ap_firmware

Radar-Visualisierung (unifi_radar_data.py)

Ein eigenes Python-Script generiert eine SVG-basierte Radar-Darstellung aller mit dem UniFi-Netzwerk verbundenen und in der Nähe befindlichen Clients. Das Radar zeigt die Signalstärke als Entfernung vom Zentrum und unterscheidet zwischen verbundenen (Connected) und lediglich sichtbaren (Nearby) Geräten.

Funktionsweise

  1. Das Script verbindet sich mit der UniFi Controller API
  2. Es fragt die aktuelle Client-Liste und Nearby-Devices ab
  3. Für jeden Client wird die Signalstärke (RSSI) ausgelesen
  4. Die Daten werden in eine SVG-Radar-Grafik umgewandelt
  5. Die SVG-Datei wird als lokale Datei gespeichert und in Home Assistant angezeigt

Script-Konfiguration

# unifi_radar_data.py - Konfiguration
# Dieses Script wird als shell_command oder über einen Cron-Job ausgeführt

UNIFI_HOST = "https://192.168.1.1"     # UniFi Controller URL
UNIFI_USER = "admin"                    # UniFi Benutzername
UNIFI_PASS = ""                         # Aus Environment-Variable
UNIFI_SITE = "default"                  # Site-Name

OUTPUT_FILE = "/config/www/unifi_radar.svg"  # Ausgabedatei

# Radar-Parameter
RADAR_RINGS = 4                         # Anzahl konzentrischer Ringe
RADAR_SIZE = 400                        # SVG-Größe in Pixel
CONNECTED_COLOR = "#4CAF50"             # Grün für verbundene Clients
NEARBY_COLOR = "#FF9800"                # Orange für Nearby-Clients
BACKGROUND_COLOR = "#1a1a2e"            # Dunkler Hintergrund

SVG-Radar Ausgabe

# Das generierte SVG zeigt:
# - Konzentrische Ringe: Signalstärke (-30dBm bis -90dBm)
# - Grüne Punkte: Verbundene Clients (Connected)
# - Orange Punkte: Sichtbare Geräte in der Nähe (Nearby)
# - Labels: Gerätename und Signalstärke
# - Kreuzlinien zur Orientierung

# Einbindung im Dashboard:
type: picture
image: /local/unifi_radar.svg
tap_action:
  action: none

MikroTik Switches

Im Netzwerk werden drei MikroTik Managed Switches eingesetzt, die über SNMP (Simple Network Management Protocol) überwacht werden. Die Switches bilden das kabelgebundene Backbone des Heimnetzwerks.

Switch-Übersicht

Modell Standort Ports Besonderheiten
CSS326-24G-2S+ Keller (Hauptverteiler) 24x 1GbE + 2x SFP+ Zentraler Switch, PoE-fähig
CSS326-24G-2S+ Arbeitszimmer 24x 1GbE + 2x SFP+ Desktop-Switch für Arbeitsplatz
CRS305-1G-4S+ Keller (10G-Backbone) 1x 1GbE + 4x SFP+ 10 Gbit/s Uplinks zwischen Switches

SNMP-Konfiguration auf MikroTik

SNMP muss auf jedem MikroTik Switch aktiviert werden. Dies erfolgt über die MikroTik Web-Oberfläche (WebFig) oder per SSH: 

# MikroTik RouterOS - SNMP aktivieren
/snmp set enabled=yes contact="admin" location="Keller"
/snmp community set 0 name=public read-access=yes write-access=no

# SNMP-Einstellungen prüfen
/snmp print
/snmp community print

SNMP Monitoring Script (snmp_switch_ports.py)

Ein eigenes Python-Script liest über SNMP die Port-Statistiken aller MikroTik Switches aus und stellt sie als Sensoren in Home Assistant bereit. Das Script ermittelt pro Port: Link-Status, Geschwindigkeit, TX/RX Bytes und Fehlerrate. 

# snmp_switch_ports.py - Auszug
import subprocess
import json

# SNMP OIDs für Interface-Monitoring
OIDS = {
    "ifDescr":      "1.3.6.1.2.1.2.2.1.2",     # Port-Beschreibung
    "ifOperStatus":  "1.3.6.1.2.1.2.2.1.8",     # Betriebsstatus (1=up, 2=down)
    "ifSpeed":       "1.3.6.1.2.1.2.2.1.5",     # Geschwindigkeit
    "ifInOctets":    "1.3.6.1.2.1.2.2.1.10",    # Empfangene Bytes
    "ifOutOctets":   "1.3.6.1.2.1.2.2.1.16",    # Gesendete Bytes
    "ifInErrors":    "1.3.6.1.2.1.2.2.1.14",    # Empfangsfehler
    "ifOutErrors":   "1.3.6.1.2.1.2.2.1.20",    # Sendefehler
}

SWITCHES = {
    "keller":        {"host": "192.168.1.10", "community": "public"},
    "arbeitszimmer": {"host": "192.168.1.11", "community": "public"},
    "backbone":      {"host": "192.168.1.12", "community": "public"},
}

def snmp_walk(host, community, oid):
    """Führt einen SNMP Walk aus und gibt die Ergebnisse zurück."""
    cmd = f"snmpwalk -v2c -c {community} {host} {oid}"
    result = subprocess.run(cmd.split(), capture_output=True, text=True)
    return parse_snmp_output(result.stdout)

def get_port_status(host, community):
    """Ermittelt den Status aller Ports eines Switches."""
    ports = {}
    descriptions = snmp_walk(host, community, OIDS["ifDescr"])
    statuses = snmp_walk(host, community, OIDS["ifOperStatus"])
    speeds = snmp_walk(host, community, OIDS["ifSpeed"])

    for idx in descriptions:
        ports[idx] = {
            "name": descriptions[idx],
            "status": "up" if statuses.get(idx) == "1" else "down",
            "speed": int(speeds.get(idx, 0)) // 1000000,  # Mbps
        }
    return ports

# Ergebnis als JSON für Home Assistant REST-Sensor
if __name__ == "__main__":
    result = {}
    for name, config in SWITCHES.items():
        result[name] = get_port_status(config["host"], config["community"])
    print(json.dumps(result, indent=2))

Cisco 3560C Switches

Zusätzlich zu den MikroTik Switches sind zwei Cisco Catalyst 3560C Layer-3 Switches im Einsatz. Diese verwalten die WiFi-Infrastruktur (Access Points, IoT-Geräte) und werden ebenfalls per SNMP überwacht. Die Switches laufen auf IOS 15.2(2)E6 und bieten SSH, Telnet und Web-Management.

Switch-Übersicht

Hostname IP-Adresse Ports Funktion Zugang
WiFi-AZ 192.168.178.193 10x 1GbE WiFi APs Arbeitszimmer SSH, Telnet, HTTP, HTTPS
WiFi-IoT 192.168.178.194 10x 1GbE IoT-Geräte VLAN SSH, Telnet, HTTP, HTTPS

SSH auf Cisco IOS aktivieren (Step-by-Step)

Falls SSH auf einem Cisco Switch noch nicht aktiviert ist, kann dies über eine Telnet-Session konfiguriert werden. Voraussetzung: ein lokaler Benutzer muss bereits existieren. 

# 1. Per Telnet verbinden
telnet 192.168.178.193

# 2. Login mit lokalem User
Username: admin
Password: ****

# 3. Enable-Modus
enable
Password: ****

# 4. Konfiguration
configure terminal

# 5. Hostname und Domain setzen (nötig für RSA Key)
hostname WiFi-AZ
ip domain-name paulis.lan

# 6. RSA-Schlüssel generieren (2048 Bit)
crypto key generate rsa modulus 2048
# → Dauert ca. 10 Sekunden

# 7. SSH Version 2 aktivieren
ip ssh version 2

# 8. VTY-Lines für SSH konfigurieren
line vty 0 15
 transport input ssh telnet
 login local
end

# 9. Konfiguration speichern
write memory

# 10. Verifizieren
show ip ssh
# → Ausgabe: "SSH Enabled - version 2.0"
Tipp: Für die Automatisierung kann ein expect-Script verwendet werden, das die Telnet-Session skriptet. Beispiel unter /config/scripts/enable_ssh_cisco.exp.

SNMP-Konfiguration auf Cisco IOS

# Cisco IOS - SNMP aktivieren
configure terminal
snmp-server community s3cr3t ro
snmp-server location "Arbeitszimmer"
snmp-server contact "admin"
end
write memory

# Verfügbare SNMP OIDs (Cisco-spezifisch):
# Uptime:     .1.3.6.1.2.1.1.3.0        (sysUpTime, Timeticks)
# CPU-Last:   .1.3.6.1.4.1.9.9.109.1.1.1.1.7.1  (cpmCPUTotal5minRev)
# Interfaces: .1.3.6.1.2.1.2.2.1.2.101xx        (Cisco nutzt 10101+ für GigE-Ports)

snmp_switch_ports.py (MikroTik + Cisco)

Das Port-Monitoring-Script wurde erweitert, um sowohl MikroTik als auch Cisco Switches zu unterstützen. Es erkennt automatisch den Switch-Typ anhand der Interface-Indizes: 

# Aufruf:
python3 snmp_switch_ports.py <host> [community] [num_ports]

# MikroTik (Standard): sequentielle Indices 1..26, Community "public"
python3 snmp_switch_ports.py 192.168.178.165

# Cisco: Indices 10101..101xx für GigabitEthernet-Ports
python3 snmp_switch_ports.py 192.168.178.193 s3cr3t 10

# Auto-Erkennung: Das Script prüft Index 10101 auf "Ethernet"
# → Cisco: GigabitEthernet0/1 bis GigabitEthernet0/10
# → MikroTik: Port 1 bis Port 26

# Ausgabe (JSON):
{
  "count": 7,
  "ports": [
    {"port": 10101, "name": "GigabitEthernet0/1", "speed": 1000,
     "in": 1.2, "out": 0.8, "util": 0, "bar": "░░░░░░░░░░"}
  ]
}

Network Admin Page

Die Network Admin Page ist eine automatisch generierte HTML-Seite, die alle Netzwerkgeräte mit ihren verfügbaren Diensten (HTTP, SSH, Telnet etc.) in einer übersichtlichen Tabelle darstellt. Die Daten stammen aus der AdGuard Home DNS API und einem parallelen Port-Scan.

Die Network Admin Page mit Infrastruktur-Geräten und entdeckten Clients

Architektur

Komponente Beschreibung
Script /config/scripts/network_admin.py
Ausgabe /config/www/network-admin.html
URL http://<HA-IP>:8123/local/network-admin.html
Datenquellen AdGuard Home API + TCP Port-Scan
Aktualisierung Alle 5 Minuten via HA command_line Sensor
Laufzeit ~5-10 Sekunden (parallelisierter Scan)

Setup Step-by-Step

  1. AdGuard Home API-Zugang sicherstellen

    Die API wird über Basic Auth angesprochen. Zugangsdaten in network_admin.py konfigurieren.

    # Test:
curl -u user:pass http://<adguard-ip>/control/clients
  2. Script anlegen 
    # /config/scripts/network_admin.py
# Konfigurierbare Parameter am Anfang des Scripts:
ADGUARD_URL  = "http://<adguard-ip>"
ADGUARD_USER = "user"
ADGUARD_PASS = "pass"         # oder aus secrets.yaml
OUTPUT_FILE  = "/config/www/network-admin.html"
SCAN_PORTS   = [22, 23, 80, 443, 4444, 5000, 5001, 8080, 8123, 8291, 8443, ...]
PORT_TIMEOUT = 0.5            # Sekunden pro Port
MAX_SCAN_WORKERS = 50         # Parallele Threads
  3. Infrastruktur-Geräte definieren

    Im Script werden bekannte Geräte mit festen Einträgen definiert (Name, IP, Gruppe, SSH-User). Diese erscheinen immer oben in eigenen Sektionen. Zusätzlich kann ein KNOWN_SERVICES Dict definiert werden, das bekannte Ports pro IP enthält – diese werden auch angezeigt, wenn der Port-Scan das Gerät nicht erreicht.

  4. HA Sensor konfigurieren 
    # packages/network_switches.yaml
command_line:
  - sensor:
      name: "Network Admin Update"
      unique_id: network_admin_update
      command: "python3 /config/scripts/network_admin.py"
      value_template: "{{ value }}"
      scan_interval: 300      # alle 5 Minuten
      command_timeout: 120    # großzügiger Timeout für Scan
      icon: mdi:web
  5. HA neu laden

    Entwicklerwerkzeuge → YAML → Alle YAML-Konfiguration neu laden

  6. Seite testen 
    # Im Browser öffnen:
http://<HA-IP>:8123/local/network-admin.html

Features der Admin Page

  • Infrastruktur-Sektionen: Router, Firewalls, MikroTik Switches, Cisco Switches, Access Points, Server – jeweils mit Status-Dot und Service-Badges
  • Entdeckte Geräte: Automatisch aus AdGuard DNS (DHCP, ARP, rDNS) mit Suchfilter
  • Klickbare Service-Links:
    • HTTP/HTTPS → öffnet im Browser
    • SSH → ssh://user@ip (Chrome Secure Shell Extension oder OS-Handler)
    • Telnet → telnet://ip
    • Spezial-Ports: HA (8123), Node-RED (8443), WinBox (8291), DSM (5000/5001), Homarr (7575), Admin (4444)
  • Auto-Refresh: Seite aktualisiert sich alle 5 Minuten
  • Dark Theme: Passt zu Homarr und HA Dark Mode
  • Stats-Übersicht: Anzahl Infrastruktur-Geräte, entdeckte Geräte, Online-Geräte

Homarr-Einbettung

  1. Homarr öffnen (z.B. http://nas:7575)
  2. Board bearbeiten → Widget hinzufügen → iframe
  3. URL: http://<HA-IP>:8123/local/network-admin.html
  4. Scrolling erlauben, Höhe anpassen (empfohlen: 800px+)

Chrome Secure Shell Extension

Damit ssh://-Links direkt im Browser funktionieren, wird die Chrome Secure Shell Extension (von Google) empfohlen. Nach Installation öffnen SSH-Links direkt eine Terminal-Session im Browser. Alternativ: PuTTY, Termius oder das System-Terminal als SSH-Handler registrieren.

Fritz!Box Integration

Die Fritz!Box dient als Internet-Gateway und wird über die native Home Assistant Integration überwacht. Sie liefert Informationen über die Internetverbindung, verbundene Geräte und den DSL-Status.

Integration einrichten

  1. Einstellungen → Geräte & Dienste → Integration hinzufügen
  2. Nach AVM FRITZ!Box suchen
  3. Host: IP-Adresse der Fritz!Box (Standard: 192.168.178.1)
  4. Benutzername: Fritz!Box Admin-Benutzer
  5. Passwort: Fritz!Box Passwort aus secrets.yaml
